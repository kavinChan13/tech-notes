AI 工程师 30/60/90 学习路径

3 个月 ≈ 280 小时 的体系化学习地图。不是"列书单"，是"按周排好课的训练营计划"。
Month 1 打通用底座 → Month 2 按目标岗位分 3 轨深度 → Month 3 冲刺面试。每月配 flagship 项目，让你简历从空白到有亮点。

怎么用这个路径 · 别只读 · 要执行

前置阅读 · 关键原则 · 时间预算
🎬生活类比 · 学 AI 像健身房训练计划
很多人想"自学 AI"，结果就是买了 10 本书，每本读了 2 章，3 个月后还是新手。

这就像去健身房 — 没有训练计划，每次都凭感觉练，3 个月毫无进步。但如果你拿到一份"周一胸 / 周二背 / 周三腿 / 每个动作 4 组 8 次"的计划，3 个月真能脱胎换骨。

本文就是 AI 学习的"训练计划"：① 每天 90 分钟 ② 按周排好课 ③ 每月 checkpoint 自检 ④ 配 flagship 项目落地。不要把它当书读，要打印出来当任务清单执行。
前置阅读：开始前请先读 X14 AI 岗位图谱 + JD 解读。至少看 §2（6 类岗位画像） + §3（能力矩阵自评）+ §9（转岗路径），确定自己走哪条路再回来执行本路径。

4 条关键原则

① 不抢跑 · 按依赖走

X06 没看完不要碰 Y02。原理是 Infra 的前提。本路径 Month 1 必须按顺序，Month 2 才能按兴趣调。

② 不装会 · 能讲才算会

"读完了" ≠ "会了"。能 30 分钟给同事讲清楚 + 答 3 个追问，才算会。读完每章建议讲一遍给自己。

③ 不积压 · 每天 90 min

连续 90 天每天 90 min >> 周末突击 5 h。大脑需要睡眠固化记忆，碎片学习更高效。

④ 必动手 · flagship 项目

读 10 篇文档 < 做 1 个 flagship。简历上写得出的 case 全部来自动手项目，不来自阅读量。

本路径默认对标：5+ 年 C++ / 系统编程 / 后端经验，3 个月内想转 AI 工程师方向。如果你不是这个画像，先在 §ch1 自我定位完成后看路径调整建议。

Daily 90 min 习惯 · 每天怎么排

关键是天天，不是猛刷
🎬生活类比 · 学 AI 像每天背单词
每天背 20 个新词，连续 100 天 = 2000 词。
一周突击 1 天背 200 个，连续 14 周 = 同样 2000 词。

看似总量一样，但实际效果差 5-10 倍。大脑需要睡眠固化记忆，每天少量 + 间隔重复，是脑科学验证的最强学习方法。
🌅 07:00-07:45
读文档 / paper（脑子最清醒，啃硬骨头） 45 min
🍱 12:30-12:50
过 5-8 张 AI 面试卡（碎片时间，复习用） 20 min
🌙 21:00-21:30
动手（写 kernel / 跑 SFT / 调 RAG） 30 min
🗓 周末
flagship 项目集中冲（4-6 h / 天） 2 天
4 条关键原则：
  1. 天天比偶尔猛刷重要（连续 90 天每天 90 min >> 周末突击 5h）
  2. 动手比读重要（Y02 vLLM 读了就跑一遍 demo；X08 RAG 读了就搭一个小 RAG）
  3. 输出比输入重要（每篇深度文读完，写 200 字笔记给一周后的自己看）
  4. 反馈比进度重要（每月 mock + 找朋友 review，比按计划读完更有效）
06

🚀 2 个 flagship 项目 · 简历最顶部那种

读 10 篇文档 < 做 1 个 flagship · 这是面试官真在意的
🎬生活类比 · flagship 像厨师的招牌菜
应聘高级厨师，面试官不会问"你看过多少菜谱"。他只问一个问题："你的招牌菜是什么？给我讲讲做法 + 客人反馈 + 哪次失败"。

AI 面试一模一样。面试官不在乎你看过多少 paper，只关心你的 flagship 项目是什么 + 数字 + 失败 case。

没有 flagship 的简历 = 没有招牌菜的厨师。本路径强制要求做 2 个。

Flagship 1 · 通用项目（Month 1 末启动，~30 h）

所有轨道通用。目标：跑通从 0 到 1 的完整 LLM 应用，理解 stack。

F1-A · 个人知识库 RAG

把过去 3 年的笔记/文档接入 RAG，做个能搜+总结的私人助手。包含：chunking 策略对比 / embedding 选 3 种对比 / hybrid search / rerank / 简单 eval

⭐ B 轨首选 · A/C 轨建议

F1-B · Code Review Agent

给一个 GitHub PR 链接，自动 review 输出 markdown 报告。包含：fetch PR → 拆解 diff → 分块 review → 汇总。可以用 MCP 范式

所有轨适用

F1-C · 简历 AI 助手

上传简历 + JD，给出匹配度 + 改进建议 + 模拟 1 轮面试问答。展示 RAG + structured output + 多轮对话

B 轨 / AI PM 转岗

Flagship 2 · 轨道专属（Month 2 推进，~16-18 h）

⭐ A 轨 · 给 vLLM / SGLang / 类似项目提 PR
  • 怎么找题：vLLM / SGLang / lmdeploy / Modelscope 的 GitHub issues，筛 "good first issue"
  • 推荐类型：① 修一个小 bug ② 加一个小 feature ③ 性能优化（小 kernel 优化）
  • 不必合并：提了就行，简历可以写 "open PR to vLLM #XXXX, addressing Y"
  • 替代：写 mini vLLM clone（核心 500 行），开源到 GitHub，详细 README
B 轨 · 上线一个真用户用的 AI 产品
  • 最简版：F1 的 RAG / Agent 真的部署上线，发到小红书 / V2EX / 推特，拉到 100 个真用户
  • 必含：cost 追踪 + eval pipeline + 至少 1 次基于用户反馈的迭代
  • 简历表述："独立设计并上线 XXX，DAU 100+，月成本 X 元，accuracy 60% → 85%"
C 轨 · 复现一篇 paper 的关键实验
  • 推荐 paper：DeepSeek-V3 MLA / GRPO / DPO / FlashAttention-2 — 任选一个
  • 不必全复现：复现核心算法 + 小规模数据集验证关键 metric 即可
  • 必含：完整 README + 实验记录 + 跟原论文差异分析 + 开源到 GitHub
  • 替代：写一篇高质量 technical blog 拆解 paper（要有自己实验或推导）

FAQ · 高频疑问

7 个最常问的问题
Q1我 C++ 5 年，但 Python 一般，能走 A 轨吗？
答
能。Python 在 AI Infra 用得相对少（核心代码大量是 C++/CUDA），Python 主要用作 binding 和测试。1-2 周熟悉一下就够。
Q2我没有 GPU，怎么动手？
本地
MacBook 用 mlx 跑小模型；Windows/Linux 用 CPU 也能跑 0.5B-1.5B 模型做学习
云
Colab Pro（$10/月，T4 GPU）/ Kaggle Notebook 免费 / 国内 AutoDL（按小时 1-3 元）/ Modal Labs / Lambda Labs
实在没有
纯学理论 + 用 API 也能撑过 Month 1 + B 轨大部分内容
Q3要不要先把数学补好再开始？
不要
X15 数学速查是按需查阅用的，不是"先修课"。如果你跟着 X06 / X07 走，需要的数学会自然提示你查。停下来补数学是常见的拖延陷阱。
Q4路径里写的"读完 X06"是什么标准？
3 档
① 看完了 = 滑过一遍，知道讲了什么 — 不算
② 读懂了 = 能复述大纲 + 关键概念 — 入门
③ 读会了 = 能 30 min 给同事讲清楚，能答 3 个追问 — 这才是"读完了"
本路径标准
以 ③ 为标准。读完每章建议尝试给自己讲一遍（或写 200 字笔记）。
Q5必须按顺序读吗？
答
Month 1 必须按顺序（X14 → X01 → X06 → X07/X12 → X08）— 有强依赖。Month 2 可以根据兴趣调，Month 3 完全自由。
Q6投简历的节奏？
Month 1 末
更新 LinkedIn / 脉脉，让朋友知道你在转型（不主动投）
Month 2 末
投 3-5 家"低风险"公司练手（不在你最想去的列表里）
Month 3 中后期
开始投目标公司，3 个月路径走完后第 1-2 周是面试转化最高的窗口
Q7我同时学 A 轨 + B 轨可以吗？
不推荐
Month 2 同时 2 轨 = 都做半吊子，面试时候被两边的深度问题问穿。先深一条，第二条 6 个月后再补。
