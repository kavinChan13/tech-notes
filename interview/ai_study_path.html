01

怎么用这个路径

  • 读法：① 先看 §前置 自我定位（15 min）→ ② 按月推进 → ③ 每月末做 checkpoint 自检
  • 时间：3 个月每周 22-25 h，每天 90 min 专注学习。3 个月 = ~280 h，体系化 AI 转型所需最小投入。
  • 原则：
    1. 不抢跑。X06 没看完不要碰 Y02 — 原理是 Infra 的前提。
    2. 每章读完做 5 个题。配套的 I06 卡片应用每章都有过滤标签，可以筛出来练。
    3. 每月一个 flagship 项目，比读 10 篇文档更有面试价值。
    4. 不要装会。读完了 ≠ 会了。能 30 分钟给同事讲清楚 + 答 3 个追问，才算会。

Daily 90 min 习惯（每天怎么排）

时段内容时长
早上 7:00-7:45读文档 / paper（脑子最清醒，吃硬骨头）45 min

关键原则：
  1. 天天比偶尔猛刷重要。连续 90 天每天 90 min >> 周末突击 5 h
  2. 动手比读重要。Y02 vLLM 内部读了就跑一遍 demo；X08 RAG 读了就搭一个小 RAG
  3. 输出比输入重要。每篇深度文读完，写 200 字笔记（给自己一周后看）
  4. 反馈比进度重要。每月 mock + 找朋友 review，比按计划读完更有效

Flagship 1 · 通用项目（Month 1 末启动，~30 h）

所有轨道通用。目标：跑通从 0 到 1 的完整 LLM 应用，理解 stack。

选项内容适合
F1-A · 个人知识库 RAG 把你过去 3 年的笔记/文档接入 RAG，做个能搜+总结的私人助手。
包含：chunking 策略对比 / embedding 选 3 种对比 / hybrid search / rerank / 简单 eval（你自己当 judge）		 B 轨首选 · A/C 轨也建议
F1-B · Code Review Agent 给一个 GitHub PR 链接，自动 review 输出 markdown 报告。包含：fetch PR → 拆解 diff → 分块 review → 汇总。可以用 MCP 范式 所有轨
F1-C · 简历 AI 助手 上传简历 + JD，给出匹配度 + 改进建议 + 模拟 1 轮面试问答。展示 RAG + structured output + 多轮对话 B 轨 / AI PM 转岗

Flagship 2 · 轨道专属（Month 2 推进，~16-18 h）

A 轨 · 给 vLLM / SGLang / 类似项目提 PR

  • 怎么找题：vLLM / SGLang / lmdeploy / Modelscope 的 GitHub issues，筛 "good first issue" 标签
  • 推荐类型：① 修一个小 bug（文档错误 / 边界 case）② 加一个小 feature（支持新模型 / 新参数）③ 性能优化（小 kernel 优化）
  • 不必合并：提了就行，简历可以写 "open PR to vLLM #XXXX, addressing Y"。被合并是加分
  • 替代方案：写一个 mini vLLM clone（核心 500 行），开源到 GitHub，详细 README

B 轨 · 上线一个真用户用的 AI 产品

  • 最简版：F1 的 RAG / Agent 真的部署上线，发到小红书 / V2EX / 推特，拉到 100 个真用户
  • 必含：cost 追踪 + eval pipeline + 至少 1 次基于用户反馈的迭代
  • 简历表述："独立设计并上线 XXX，DAU 100+，月成本 X 元，accuracy 从 60% 调到 85%"

C 轨 · 复现一篇 paper 的关键实验

  • 推荐 paper：DeepSeek-V3 MLA / GRPO / DPO / FlashAttention-2 — 任选一个
  • 不必全复现：复现核心算法 + 在小规模数据集上验证关键 metric 即可
  • 必含：完整 README + 实验记录 + 跟原论文的差异分析 + 开源到 GitHub
  • 替代：写一篇高质量 technical blog 拆解 paper（要有自己的实验或推导）

FAQ

Q1：我 C++ 5 年

Q1：我 C++ 5 年，但 Python 一般，能走 A 轨吗？

能。Python 在 AI Infra 用得相对少（核心代码大量是 C++/CUDA），Python 主要用作 binding 和测试。1-2 周熟悉一下就够。

Q2：我没有 GPU，怎么动手？

  • 本地：MacBook 用 mlx 跑小模型；Windows/Linux 用 CPU 也能跑 0.5B-1.5B 模型做学习
  • 云：Colab Pro（$10/月，T4 GPU）/ Kaggle Notebook 免费 / 国内 AutoDL（按小时 1-3 元）/ Modal Labs / Lambda Labs
  • 实在没有：纯学理论 + 用 API 也能撑过 Month 1 + B 轨大部分内容

Q3：要不要先把数学补好再开始？

不要。X15 数学速查是按需查阅用的，不是"先修课"。如果你跟着 X06 / X07 走，需要的数学会自然提示你查。停下来补数学是常见的拖延陷阱。

Q4：路径里写的"读完 X06"是什么标准？

三档：
① 看完了 = 滑过一遍，知道讲了什么 — 不算
② 读懂了 = 能复述大纲 + 关键概念 — 入门
③ 读会了 = 能 30 min 给同事讲清楚，能答 3 个追问 — 这才是"读完了"
本路径以 ③ 为标准。读完每章建议尝试给自己讲一遍（或写 200 字笔记）。

Q5：必须按顺序读吗？

Month 1 必须按顺序（X14 → X01 → X06 → X07/X12 → X08）— 有强依赖。Month 2 可以根据兴趣调，Month 3 完全自由。

Q6：投简历的节奏？

  • Month 1 末：更新 LinkedIn / 脉脉，让朋友知道你在转型（不主动投）
  • Month 2 末：投 3-5 家"低风险"公司练手（不在你最想去的列表里）
  • Month 3 中后期：开始投目标公司，3 个月路径走完后第 1-2 周是面试转化最高的窗口

Q7：我同时学 A 轨 + B 轨可以吗？

不推荐。Month 2 同时 2 轨 = 都做半吊子，面试时候被两边的深度问题问穿。先深一条，第二条 6 个月后再补。
