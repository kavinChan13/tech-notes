auto / decltype 类型推导

让编译器替你写类型，尤其在模板/泛型场景中是革命性改进。

C++11

auto / decltype — 类型推导

让编译器替你写类型，尤其在模板/泛型场景中是革命性改进。

核心概念（面试表达）

auto：让编译器从初始化表达式 推导 变量类型，规则与函数模板参数推导一致（会去掉引用和顶层 const）。

decltype(expr)：在不求值表达式的前提下推导其类型，保留引用 / const / cv 限定符，是写转发包装、SFINAE 的基础工具。

编译期角度：两者都是 编译期 行为，运行时零开销，最终生成的目标代码与手写类型完全一致。

为什么引入

  • 模板里类型名极长：std::map<K,V>::const_iterator
  • Lambda 类型不可写出（每个 lambda 都是匿名 closure 类型）
  • 函数返回类型依赖参数类型 → 必须 decltype + 尾置返回
  • 替代旧写法：迭代器 typename Container::iterator it = ...

标准示例

#include <vector>
#include <map>

void demo() {
    std::vector<int> v{1,2,3};
    auto it = v.begin();           // std::vector<int>::iterator
    const auto& ref = v[0];          // const int&

    auto lambda = [](int x){ return x*2; };  // 匿名 closure

    // decltype 保留 cv 和引用
    int x = 0; const int& r = x;
    decltype(r) y = x;                // const int&

    // 尾置返回类型 + decltype（C++14 起可省略）
    auto add = [](auto a, auto b) -> decltype(a+b) { return a+b; };
}

    using Handler = std::function<Reply(const Request&)>;
    std::unordered_map<std::string, Handler> handlers_;
public:
    void dispatch(const std::string& method, const Request& req) {

        auto it = handlers_.find(method);          // ✅ 简洁
        if (it == handlers_.end()) throw NotFound{method};
        const auto& handler = it->second;            // 不拷贝
        auto reply = handler(req);                  // 类型由返回值推导
        send_reply(std::move(reply));
    }
};

template<class F, class... Args>
auto measure(F&& f, Args&&... args)
    -> decltype(f(std::forward<Args>(args)...))   // 精确转发返回类型
{
    auto t0 = std::chrono::steady_clock::now();
    auto&& ret = std::forward<F>(f)(std::forward<Args>(args)...);
    LOG_INFO("latency_us={}", (std::chrono::steady_clock::now()-t0).count());
    return std::forward<decltype(ret)>(ret);
}

C++11

nullptr — 类型安全的空指针

核心概念

nullptr 是一个 std::nullptr_t 类型的字面量，可隐式转换到任意指针类型，但不能转换为整型。

解决了 C 时代 NULL（通常 #define NULL 0）在函数重载下的二义性 bug。

为什么引入

  • NULL 是 0 → 选错重载（int 版本 vs 指针版本）
  • 模板里 NULL 退化为 int → 编译失败或行为错误
  • 需要一个真正的「指针字面量」类型

对比：NULL 的 bug

void f(int);
void f(char*);

f(NULL);     // ❌ 调用 f(int)！NULL 通常是 0
f(nullptr);  // ✅ 调用 f(char*)

Range-based for + 统一初始化 {}

标准示例 · REAL工程中的使用方式 · 配置加载 + 连接遍历

C++11

Range-based for & 统一初始化 {}

Range-based for 本质

编译器把 for (auto x : c) 改写为：调用 begin(c) / end(c)，循环 ++it 并解引用。

支持任意提供 begin/end 的类型（含 C 数组、std 容器、自定义容器）。

统一初始化 {} (Uniform initialization)

用 {} 初始化任何对象（聚合、容器、POD、有构造函数的类、返回值）。

禁止窄化转换（int x{3.14} 编译错），更安全。

引入 std::initializer_list<T> 让自定义类支持 MyVec v{1,2,3};

标准示例

std::vector<int> v{1,2,3,4};      // 容器 = initializer_list 构造
int arr[]{10,20,30};

for (auto x : v)         { /* 拷贝 */ }
for (auto& x : v)        { x *= 2; }    // 修改
for (const auto& x : v)  { /* 只读，避免拷贝 */ }

struct Point { int x, y; };
Point p{1, 2};                    // 聚合初始化
std::pair<int,std::string> kv{1, "hello"};

int n{3.14};                      // ❌ 编译错：窄化转换

Lambda 表达式

capture 写法对照 · 标准示例 · REAL工程中的使用方式 · 线程池 / 异步任务 / Reactor 回调

C++11

Lambda 表达式 — 内联匿名函数对象

核心概念

语法 [capture](params) -> ret { body }。每个 lambda 是一个编译器自动生成的匿名类（closure type），含有 operator()。

capture 列表决定如何把外层变量「装进闭包对象的成员」。

mutable 让按值捕获的成员在 operator() 中可修改。

为什么引入

  • 替代笨重的「自定义 functor 类」
  • 替代 std::bind（编译更快、可读性高）
  • STL 算法 (std::sort, for_each) 终于变好用
  • 异步/线程库需要传递可调用对象

capture 写法对照

写法含义典型场景
[]不捕获纯函数 / sort 比较器
[=]所有用到的外层变量按值捕获异步任务（变量副本生命周期独立）
[&]所有按引用捕获同步算法、必须保证外层变量存活
[x, &y]x 按值，y 按引用精确控制最常用
[this]捕获 this 指针类成员函数内部 lambda
[*this] (C++17)按值拷贝整个对象异步任务跨对象生命周期

标准示例

std::vector<int> v{3,1,4,1,5};
std::sort(v.begin(), v.end(), [](int a, int b){ return a > b; });

int threshold = 3;
auto cnt = std::count_if(v.begin(), v.end(),
    [threshold](int x){ return x > threshold; });

// mutable 示例
auto counter = [n = 0]() mutable { return ++n; };  // C++14 init capture
counter(); counter();

智能指针 unique_ptr / shared_ptr / weak_ptr

现代 C++ 内存管理的基石。三种语义：独占 / 共享 / 观察。

C++11

智能指针 — RAII 资源所有权

现代 C++ 内存管理的基石。三种语义：独占 / 共享 / 观察。

三种语义

  • unique_ptr<T>：独占所有权，不可拷贝，可移动；零开销（≈ 裸指针 + 析构调用）。
  • shared_ptr<T>：引用计数共享所有权；控制块 + 强/弱计数（原子操作）。
  • weak_ptr<T>：观察者，不影响对象生命周期；用 lock() 临时升级为 shared_ptr。

为什么引入

  • 裸 new/delete → 内存泄漏 / 双重释放
  • 异常路径下手动管理几乎无解
  • 取代 boost::shared_ptr 进入标准
  • auto_ptr（C++98）语义混乱已废弃

开销与控制块（面试高频）

unique_ptr
sizeof = sizeof(T*)
无堆控制块
shared_ptr
sizeof = 2 个指针
堆上控制块 (强/弱计数原子)
make_shared
对象 + 控制块
一次堆分配（更快）

标准示例

auto u = std::make_unique<Buffer>(4096);  // C++14；C++11 用 unique_ptr<...>(new ...)
auto s = std::make_shared<Session>(fd);
std::weak_ptr<Session> w = s;

if (auto sp = w.lock()) {     // 安全升级
    sp->send("hi");
} else {
    LOG_INFO("session expired");
}

// 自定义 deleter：管理 C 资源
auto close_fd = [](int* p){ ::close(*p); delete p; };
std::unique_ptr<int, decltype(close_fd)> fd_holder(new int(::open(...)), close_fd);

移动语义 + 右值引用

让「资源所有权转移」不再依赖深拷贝，是性能革命的核心。

C++11

移动语义 & 右值引用 (T&&)

让「资源所有权转移」不再依赖深拷贝，是性能革命的核心。

核心概念

左值 / 右值 / xvalue：左值有名字、可取地址；右值是临时对象。T&& 只能绑右值。

移动构造 / 移动赋值：把源对象的内部资源（堆指针 / 文件句柄）窃取过来，源对象置为「合法但未指定」状态。

std::move 不移动任何东西，它只是把左值无条件转换为右值。

为什么引入

  • 函数返回大对象 → 旧版本必须深拷贝（NRVO 之外）
  • 容器扩容 / 元素交换需要昂贵拷贝
  • std::vector<std::string> 之类的嵌套容器性能受拷贝拖累
  • 独占资源的类型（如 unique_ptr）必须能「转交」

规则五法（Rule of 5）

class Buffer {
    char* data_; size_t size_;
public:
    Buffer(size_t n)            : data_(new char[n]), size_(n) {}
    ~Buffer()                     { delete[] data_; }
    Buffer(const Buffer& o)     : data_(new char[o.size_]), size_(o.size_) {
        std::memcpy(data_, o.data_, size_);
    }
    Buffer(Buffer&& o) noexcept   : data_(o.data_), size_(o.size_) {  // ✅ 偷资源
        o.data_ = nullptr; o.size_ = 0;                       // ✅ 源置 null
    }
    Buffer& operator=(const Buffer&);    // 拷贝赋值
    Buffer& operator=(Buffer&&) noexcept; // 移动赋值
};
noexcept 很关键：std::vector 扩容时若元素的移动构造是 noexcept，就用移动；否则只能拷贝（强异常保证）→ 性能差几个数量级。

完美转发 / 万能引用

引用折叠规则速记 · 标准示例 · REAL工程中的使用方式 · 通用任务提交 + emplace

C++11

完美转发 & 万能引用 (Universal Reference)

核心概念

函数模板里的 T&& 是万能引用（forwarding reference），通过引用折叠规则既能绑左值也能绑右值。

std::forward<T>(x)：根据 T 是否带引用，决定保留左值/右值属性 → 完美转发。

编译期机制：模板参数推导 + 引用折叠（& & → &，& && → &，&& && → &&）。

为什么引入

  • 包装函数（如 emplace_back、make_shared）必须把参数原样传到底层构造函数
  • 避免「对左值参数被强制按值复制 / 对右值参数失去 move 能力」

引用折叠规则速记

实参T 推导为T&& 结果
左值 int xint&int& && → int&
右值 42intint&&

标准示例

template<class T>
void wrapper(T&& arg) {              // 万能引用
    backend(std::forward<T>(arg));     // ✅ 完美转发
}

// 错误对比
template<class T>
void bad(T&& a) {
    backend(a);                          // ❌ a 是左值（有名），右值属性丢失
    backend(std::move(a));               // ❌ 总是 move，破坏左值实参
}

REAL工程中的使用方式 · 通用任务提交 + emplace

// thread_pool.h - submit 接受任意 callable + 参数包
class ThreadPool {
public:
    template<class F, class... Args>
    auto submit(F&& f, Args&&... args)
        -> std::future<std::invoke_result_t<F, Args...>>
    {
        using R = std::invoke_result_t<F, Args...>;

        auto task = std::make_shared<std::packaged_task<R()>>(
            // 用 bind/lambda 把参数原样捕获，避免提前求值
            [f = std::forward<F>(f),
             args_tuple = std::make_tuple(std::forward<Args>(args)...)]() mutable {
                return std::apply(std::move(f), std::move(args_tuple));
            });
        auto fut = task->get_future();
        {
            std::lock_guard lk{mu_};
            tasks_.emplace([task]{ (*task)(); });
        }
        cv_.notify_one();
        return fut;
    }
};

// 容器 emplace_back 内部实现概念
template<class... Args>
void vector<T>::emplace_back(Args&&... a) {
    // 在 buf_[size_] 处原地构造，避免临时对象 + 拷贝
    new (buf_ + size_++) T(std::forward<Args>(a)...);
}
面试高频：① auto&& 也是万能引用；② const T&& 不是万能引用，只能绑 const 右值；③ 万能引用与重载并存时容易劫持所有调用 → 用 std::enable_if 或 C++20 concepts 约束。

// thread_pool.cpp - 真实生产风格
class ThreadPool {
    std::vector<std::thread> workers_;
    std::queue<std::function<void()>> tasks_;
    std::mutex mu_;
    std::condition_variable cv_;
    bool stop_{false};
public:
    explicit ThreadPool(size_t n) {
        for (size_t i = 0; i < n; ++i)
            workers_.emplace_back([this, i]{ worker_loop(i); });
    }
    ~ThreadPool() {
        { std::lock_guard lk{mu_}; stop_ = true; }
        cv_.notify_all();
        for (auto& t : workers_) if (t.joinable()) t.join();
    }
private:
    void worker_loop(size_t id) {
        set_thread_name("pool-" + std::to_string(id));
        while (true) {
            std::function<void()> task;
            {
                std::unique_lock lk{mu_};
                cv_.wait(lk, [this]{ return stop_ || !tasks_.empty(); });
                if (stop_ && tasks_.empty()) return;
                task = std::move(tasks_.front());
                tasks_.pop();
            }
            try { task(); }
            catch (const std::exception& e) { LOG_ERROR("task ex: {}", e.what()); }
        }
    }
};
反模式：① 在锁内调用任务 → 长时间持锁；② 用 cv.wait(lk) 不带谓词；③ thread t(f); t.detach(); 后 f 引用了局部对象 → UB。

  • C++ 第一次正式定义内存模型（DR2008 → C++11）
  • 无锁数据结构必备；自旋锁 / 引用计数 / 标志位
  • 消除编译器/硬件重排带来的并发 bug

典型对比：seq_cst vs acquire/release

struct ControlBlock {

        strong.fetch_add(1, std::memory_order_relaxed);   // 加 1 不需要同步

        }

C++11

核心概念

};

struct NonCopyable {

class LogRotator {
    fs::path dir_;
    size_t max_files_;
    uint64_t max_total_bytes_;
public:
    void enforce_quota() {
        std::vector<fs::directory_entry> files;
        for (auto& e : fs::directory_iterator(dir_))
            if (e.is_regular_file() && e.path().extension() == ".log")
                files.push_back(e);

        std::sort(files.begin(), files.end(),
            [](auto& a, auto& b){ return fs::last_write_time(a) < fs::last_write_time(b); });

        uint64_t total = 0;
        for (const auto& f : files) total += f.file_size();

        while ((files.size() > max_files_ || total > max_total_bytes_) && !files.empty()) {
            total -= files.front().file_size();
            std::error_code ec;
            fs::remove(files.front().path(), ec);
            files.erase(files.begin());
        }
    }
};

Concepts — 模板的类型约束

核心概念

语法：template<Concept T> / requires 子句 / requires { 表达式; } 表达式。

标准示例

// requires 表达式校验多个能力
template<class T>
concept Serializable = requires(T t, Buffer& b) {
    { t.serialize(b)   } -> std::same_as<void>;
    { T::deserialize(b) } -> std::same_as<T>;
};

Coroutines — 无栈协程

核心概念

注意：标准只提供「机制」，实际可用的 Task / Generator 类型要业务自己写或用第三方库（cppcoro / folly::coro）。

为什么引入

  • 异步编程「回调地狱」

  • 对标 C# / Python 的 async/await

简化示例 (generator 概念)

// 业务定义的最简 generator
template<class T> struct Generator {
    struct promise_type {
        T value;
        Generator get_return_object() { return {}; }
        std::suspend_always initial_suspend() { return {}; }
        std::suspend_always final_suspend() noexcept { return {}; }
        std::suspend_always yield_value(T v) { value = v; return {}; }
        void return_void() {}
        void unhandled_exception() { std::terminate(); }
    };
    // ...
};

Generator<int> range(int n) {
    for (int i = 0; i < n; ++i) co_yield i;
}

class DiskFlusher {
    std::jthread worker_;
    std::mutex mu_;
    std::condition_variable_any cv_;
    std::queue<LogEntry> q_;
public:
    DiskFlusher() {
        worker_ = std::jthread([this](std::stop_token st){ loop(st); });
    }
    void push(LogEntry e) {
        { std::lock_guard lk{mu_}; q_.push(std::move(e)); }
        cv_.notify_one();
    }
private:
    void loop(std::stop_token st) {
        // stop_callback：stop 触发时唤醒 cv，避免长睡死等
        std::stop_callback wake(st, [this]{ cv_.notify_all(); });

        while (!st.stop_requested()) {
            std::unique_lock lk{mu_};
            cv_.wait(lk, st, [this]{ return !q_.empty(); });
            if (st.stop_requested() && q_.empty()) break;
            auto e = std::move(q_.front()); q_.pop();
            lk.unlock();
            write_to_disk(e);
        }
        // 退出前 drain 剩余
        drain_remaining();
    }
};

    s.start();
    wait_signal(SIGTERM);

}

核心用法

// 1. 无限序列（惰性）

    int i = 0;
    while (true) co_yield i++;

    std::print("{} ", x);                       // 0 1 2 ... 9

std::generator<std::string> lines(const std::filesystem::path& p) {
    std::ifstream f{p};
    std::string line;
    while (std::getline(f, line)) co_yield std::move(line);

    handle(l);

// 3. 递归 generator — 树遍历
std::generator<Node&> walk(Node& root) {
    co_yield root;
    for (auto& c : root.children)
        for (auto& n : walk(c))
            co_yield n;

Monadic optional / expected — 函数式错误处理链

C++23 给 std::optional 和 std::expected 加上了三个 monadic 算子，让"有值就继续做、没值/出错就短路"的代码无需手写嵌套 if。

三个算子

算子语义对应 Rust
and_then(f)有值 → 调 f 返回新 optional/expected；无值 → 直通flat_map / and_then
transform(f)有值 → 用 f 变换包成新 optional；无值 → 直通map

对比：嵌套 if vs monadic

// 老写法：金字塔
std::optional<User> find_user(UserId);
std::optional<Profile> load_profile(UserId);
std::optional<Avatar>  find_avatar(const Profile&);

std::optional<Avatar> get(UserId id) {
    if (auto u = find_user(id)) {
        if (auto p = load_profile(u->id)) {
            if (auto a = find_avatar(*p)) {
                return a;
            }
        }
    }
    return std::nullopt;
}

// C++23 monadic：扁平管道
std::optional<Avatar> get(UserId id) {
    return find_user(id)
        .and_then([](auto u) { return load_profile(u.id); })
        .and_then([](auto p) { return find_avatar(p); });
}

    return parse(r)
        .and_then(validate)
        .and_then(authenticate)
        .and_then(dispatch)
        .transform([](auto r) { return enrich(r); })
        .or_else([](Err e) -> std::expected<Resp, Err> {
            log("failed: {}", e);
            return std::unexpected(to_external(e));
        });
}

if consteval — 完整的编译期分支

典型用法 · 对比 C++20 写法

C++23

if consteval — "我现在是在编译期被求值吗？"

C++20 给了 std::is_constant_evaluated()，但作为函数有限制（不能在分支条件里用它来调 consteval 函数）。C++23 补 if consteval { ... } 语法消除这些限制。

典型用法

constexpr double my_sqrt(double x) {
    if consteval {
        // 编译期：用迭代公式（不能调用 std::sqrt 因它非 constexpr）
        double g = x;
        for (int i = 0; i < 20; ++i) g = (g + x / g) / 2;
        return g;
    } else {
        // 运行期：用硬件指令
        return std::sqrt(x);
    }
}

constexpr double A = my_sqrt(2);        // 编译期算
double b = my_sqrt(read_value());     // 运行期 — 走硬件

对比 C++20 写法

// C++20 - 用 is_constant_evaluated()
constexpr double my_sqrt_v20(double x) {
    if (std::is_constant_evaluated()) {
        // 但这里仍是普通 if，编译器为两个分支都要生成代码
        // 而且不能调 consteval 函数
    } else {
        return std::sqrt(x);
    }
}

// C++23 - if consteval
constexpr double my_sqrt_v23(double x) {
    if consteval {
        // 编译器把这段视为"只可能在编译期跑"
        // 可以调 consteval 函数
        return compile_time_only_helper(x);
    }
    return std::sqrt(x);
}
用例：① 编译期 / 运行期走完全不同的算法（编译期可读，运行期可快）；② 编译期断言走 static_assert，运行期走 assert；③ 编译期生成查表常量，运行期直接查。

std::hazard_pointer + std::rcu — 无锁回收终于标准化

无锁数据结构最难的不是 CAS，是"什么时候 delete 节点"。C++26 把两个工业标准方案进入标准库：

std::hazard_pointer (P1121)

#include <hazard_pointer>

struct Node : std::enable_hazard_pointer<Node> {
    T value;
    std::atomic<Node*> next;
};
std::atomic<Node*> head_;

bool pop(T& out) {
    std::hazard_pointer hp = std::make_hazard_pointer();
    Node* old;
    do {
        old = hp.protect(head_);                  // 受保护读
        if (!old) return false;
    } while (!head_.compare_exchange_weak(old, old->next));
    out = std::move(old->value);
    hp.reset_protection();
    old->retire();                                 // 延迟回收
    return true;
}

std::rcu (P2530)

#include <rcu>

struct Config : std::rcu_obj_base<Config> { int v; };
std::atomic<Config*> g_cfg;

// 读端 - 零开销（不计 atomic load）
void reader() {
    std::rcu_default_domain().lock();
    Config* c = g_cfg.load(std::memory_order_acquire);
    use(c->v);
    std::rcu_default_domain().unlock();
}

// 写端 - 异步回收
void update(int v) {
    Config* newc = new Config{{}, v};
    Config* old  = g_cfg.exchange(newc, std::memory_order_acq_rel);
    old->retire();                                // 异步 delete

Reflection (P2996) — 编译期静态反射

C++ 等了 20 年的特性。P2996 是编译期反射（不是 Java 那种运行时反射），可在编译期内省类型成员、生成 JSON 序列化、生成 ORM 映射、消除几乎所有"宏 + 模板偏特化"的样板。

核心机制：reflexpr 与 splicers

// 反射运算符 ^ 获取类型/成员的 meta-info
constexpr auto info = ^Point;                       // 反射 Point 类型

// 遍历成员变量
struct Point { int x; int y; };

constexpr std::vector members = std::meta::members_of(^Point);
// members 在编译期是 [^Point::x, ^Point::y]

// "splicer" [: r :] - 把 meta-info 还原回代码
template<class T>
void print_all(const T& obj) {
    [: std::meta::expand(std::meta::members_of(^T)) :] >> [&]<auto e>{
        std::println("{} = {}", std::meta::name_of(e), obj.[: e :]);
    };
}

Point p{1, 2};
print_all(p);   // 输出: x = 1, y = 2

杀手级应用 ① — 通用 JSON 序列化（不用宏 / 不用模板偏特化）

template<class T>
std::string to_json(const T& obj) {
    std::string out = "{";
    constexpr auto ms = std::meta::nonstatic_data_members_of(^T);
    [: ms :] >> [&]<auto m>{
        out += std::format("\"{}\":{},", std::meta::name_of(m),
                           to_json_value(obj.[: m :]));
    };
    out.pop_back(); out += "}";
    return out;
}

Point p{1, 2};
to_json(p);   // {"x":1,"y":2}

杀手级应用 ② — 自动 enum 到字符串（替代 magic_enum 库）

template<class E>
constexpr std::string_view enum_name(E v) {
    return [: std::meta::enumerators_of(^E) :] >> [&]<auto e>{
        if ([: e :] == v) return std::meta::name_of(e);
    };
}

enum class Color { Red, Green, Blue };
enum_name(Color::Green);   // "Green"

对 C++ 工程的颠覆

  • 消灭一大类宏：X_MACRO / Boost.PP 等几十年丑陋技巧告别
  • 消灭很多模板偏特化：tuple-like, hash, formatter 都能自动生成
  • 序列化 / 反序列化框架：再也不用手写 FOR_EACH_FIELD(json, name, age, ...)
  • ORM、依赖注入、命令行参数解析都将大大简化

// 今天（C++17/20） — std::visit + overloaded
using Shape = std::variant<Circle, Square, Triangle>;
double area(const Shape& s) {
    return std::visit([](auto&& x) -> double {
        using T = std::decay_t<decltype(x)>;
        if constexpr (std::is_same_v<T, Circle>)   return 3.14 * x.r * x.r;
        else if constexpr (std::is_same_v<T, Square>) return x.a * x.a;
        else                                            return 0.5 * x.b * x.h;
    }, s);
}

// C++26 inspect — 直接
double area(const Shape& s) {
    return inspect (s) {
        <Circle>   [r]       => 3.14 * r * r;
        <Square>   [a]       => a * a;
        <Triangle> [b, h]    => 0.5 * b * h;
    };
}

std::simd — 标准化的便携 SIMD

P1928 把 std::experimental::simd（已在 GCC libstdc++ 多年）转正为 std::simd。让 C++ 第一次有跨平台、跨 ISA的 SIMD 类型 — 不再写 _mm256_* intrinsics。

基本用法

#include <simd>
namespace stdx = std;

// 1. 自动选最佳宽度的 simd 类型
using floatv = std::simd<float>;       // AVX2 上是 8 lane，AVX-512 是 16 lane
floatv a{1.f}, b{2.f};
floatv c = a + b * b;                // 1 条 SIMD 指令

// 2. 显式宽度
using v8 = std::fixed_size_simd<float, 8>;

// 3. 从内存装载 / 存储
float src[1024], dst[1024];
for (size_t i = 0; i < 1024; i += floatv::size()) {
    floatv v{&src[i], std::vector_aligned};
    v = v * 2.5f + 1.0f;
    v.copy_to(&dst[i], std::vector_aligned);
}

// 4. 掩码（条件操作）
auto mask = (a > 0.f);                     // simd_mask<float>
where(mask, a) = 0.f;                     // 把正数清零
