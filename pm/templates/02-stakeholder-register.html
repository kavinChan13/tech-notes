01是什么 / 为什么

Stakeholder Register 是 PM 的"政治雷达图"。EM 时代靠职权说话，PM 时代 70% 是"无团队带项目"，这份文档决定了你能不能推得动事。

它解决 3 个问题：① 谁是关键人？（权力-利益方格）② 他/她在不在意？（立场 + 关注）③ 用什么货币换？（Cohen-Bradford 6 类）

没有 Stakeholder Register 的跨部门项目，等于在黑夜里开车 —— 你可能开到，也可能撞墙。

02字段说明

字段为什么存在 / 怎么用
姓名 + 角色具体到人 + 职级（"VP 张三"，不是"产品部"）
组织 / 部门内部 / 外部 / 客户 / 监管 — 后期沟通策略不同
Power 权力1-5 分 — 此人能否停项目 / 改方向 / 调资源
Interest 利益1-5 分 — 此项目对其团队 / KPI 影响多大
立场 Stance支持 / 中立偏支 / 中立 / 中立偏反 / 反对（5 档）
关键关注他/她在意什么？担心什么？想要什么？
贵货币Cohen-Bradford 6 类中最贵的 1-2 类（任务/地位/关系/个人/灵感/组织）
沟通频率每周 / 双周 / 月度 / 事件触发
沟通渠道面对面 / 邮件 / IM / Steering（按权力 + 偏好）
策略Q1 (Keep Satisfied) / Q2 (Manage Closely) / Q3 (Monitor) / Q4 (Keep Informed)
下次评审日期下次更新这一行的日期，避免遗忘
Insight "贵货币"是这份模板独有的字段，比 PMBOK 标准多。原因：Cohen-Bradford 6 类货币把"我能给对方什么"具体化（任务 / 地位 / 关系 / 个人 / 灵感 / 组织），让影响力从"靠关系"升级到"按工具"。

03Power-Interest 方格 4 象限策略

象限典型人策略常见动作
Q1 高权力 / 低利益CTO / CFO（没具体管你）Keep Satisfied 持续满足月度 brief，先于他们听到坏消息
Q2 高权力 / 高利益Sponsor / 关键客户Manage Closely 紧密管理每周 1on1 + 月度 Steering
Q3 低权力 / 低利益边缘支援团队Monitor 监测合并到 Weekly Update 邮件列表
Q4 低权力 / 高利益研发 IC / QA / 客服Keep Informed 充分知会让他们成为消息先知，自发支持

关键人货币策略示例（3 个最难推的）

  • Tomás (Customer Success Head)，立场中立偏反：他最贵货币是 任务 + 关系。我能给：① 给他 1 名 PM 助理在迁移期间帮他对接客户（任务）；② 把客户成功故事在 PR 里给他团队冠名（关系/地位）。换：他保证 4 个 EU 客户准时迁移。
  • Pavel (客户 X CTO)，立场中立偏反：贵货币 关系 + 任务。我能给：① 我方 3 个工程师驻场 4 周（任务）；② Lin Wei 亲自飞过去吃饭 + 签战略协议（关系）。换：他承诺 9/15 前 sandbox 接入。

⚖️DO & DON'T

✅ DO

  • 启动后 2 周内完成首版（基于 ≥ 8 个 1on1）
  • 双周 review，不是"建一次扔了"
  • 每个 stakeholder 标 1-2 类贵货币，盘点能给的具体筹码
  • 立场 / 货币字段 PM 私有，公开版仅 Top 5
  • Q1 沉睡老虎必须保持月度 brief，避免突袭

❌ DON'T

  • 不要写"产品部" / "研发部" — 必须具体到人
  • 不要把私版发到群 — 立场字段一旦泄露关系即破裂
  • 不要立场只有"支持/反对"两档 — "中立偏反"是最危险层
  • 不要按职级排序 — Power × Interest 才是真排序
  • 不要漏掉外部（监管 / 客户 / 供应商）
