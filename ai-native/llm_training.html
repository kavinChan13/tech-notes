每阶段干什么 · 你该懂到什么程度

LLM 训练的混乱在于名词太多：Pretrain / Continued Pretrain / SFT / Instruction Tuning / RLHF / DPO / GRPO / RLAIF / Constitutional AI / Process Reward... 本文先用一张图把整个流水线讲清楚，再分章节展开。

Stage 1

Pretrain

学会"语言"
数据：10-20T tokens 互联网文本
目标：next-token prediction
成本：$1M-100M+
产出：base model（不会聊天）
Stage 2

SFT

学会"对话"
数据：10k-1M 对话样本
目标：模仿示范回答
成本：$1k-100k
产出：instruct model（能聊天但可能瞎说）
Stage 3

Reward Model

学会"什么是好答案"
数据：10k-100k 人类偏好对
目标：评分 A vs B 哪个更好
成本：$10k-200k（人工标注贵）
产出：reward 评分器
Stage 4

RLHF / DPO / GRPO

学会"对齐人类偏好"
数据：Stage 3 reward model 评分
目标：让 reward 最大化
成本：$10k-1M
产出：aligned model（如 ChatGPT）
4 阶段对应 4 个能力跃迁：① Pretrain = 让模型"懂语言"，但还是"自动补全机"；② SFT = 学会"问答格式"和初步指令理解；③ RM = 把"模糊的人类偏好"变成"数字打分函数"；④ RLHF/DPO = 让模型答案朝着 RM 打分高的方向优化。
01

训练全景 · 把模型"变好"的 3 个步骤

学语言 → 学说话 → 学讨人喜欢
🎬生活类比 · 培养一个客服
想象你要培养一个客服。

Step 1 · 学中文（Pretrain）：让他读 1000 万本书 / 论坛 / 维基百科。读完之后他懂中文了，但不懂客服的话术 — 给他个客户问题，他可能给你写一首诗（自动补全模式）。

Step 2 · 学客服话术（SFT）：给他看 10000 个客服对话示例。读完他学会客服格式："您好，您的问题..."。但还是会瞎答（说错政策、推荐错产品）。

Step 3 · 学讨好客户（RLHF/DPO）：找 1000 个真实客户，每个客户给两个候选答案打分。客服根据"客户更喜欢哪个"反馈不断调整。最后变成"答得对 + 答得让客户满意"的金牌客服。

ChatGPT 就是这么被"养出来的"。
学中文 → Pretrain  ·  学话术 → SFT  ·  学评分 → Reward Model  ·  学讨好 → RLHF / DPO

1.1 Cost 分布 · 大头在哪

Pretrain（数月 训练）
$1M - $100M+
SFT（几天 训练）
$1k - $100k
RLHF/DPO（数周 训练）
$10k - $1M
Pretrain 占 90% 成本，但 SFT + RLHF 决定模型的"个性"和"对齐度"。Llama 3 70B 的 base 是同一个，但 Llama / Mistral / Qwen 用不同 SFT+RLHF 数据微调出风格迥异的版本。"模型能力"≈ Pretrain 决定，"模型好用"≈ SFT+RLHF 决定。
02

Pretrain · 烧钱学语言

Scaling Laws / 数据 curation / 训练目标
🎬生活类比 · 让孩子读 1000 万本书
想象你养个超快的孩子，能 3 个月读完互联网上所有公开文本（10-20 万亿字）。读的时候不要求他记住，只要求他玩"猜下一个字"游戏（next-token prediction）。

神奇的事：读完之后，他自动学会了语法、常识、代码、数学、几十种语言。不是因为他被教了这些，而是"猜下一个字"这个任务足够通用 — 要猜得准，必须懂上下文。

这就是 GPT 系列的"涌现能力" — 单一目标（next-token prediction）+ 大数据 + 大模型 = 通用智能涌现。

2.1 Scaling Laws · "更大就更好"的数学法则

2020 OpenAI scaling law + 2022 DeepMind Chinchilla 发现的核心规律：

📈 第一定律 · 模型越大越聪明

模型参数从 1B → 10B → 100B → 1000B，loss 按 power law 下降。10x 参数 ≈ 10% loss 降低。能力随参数指数般出现新能力（涌现）。

📊 第二定律 · Chinchilla Optimal

给定 compute 预算，最优 = 参数:数据 ≈ 1:20。即 7B 模型应该训 140B token。早期 GPT-3 175B 只训 300B token = 欠拟合。Llama 2 7B 训 2T token = 严重过拟合数据但效果好。

2.2 数据 Curation · 决定模型上限

🎬生活类比 · 给孩子读什么书
让孩子读 1000 万本书，但有 80% 是垃圾邮件 / 重复广告 / 错误内容，效果会很差。反之，挑高质量教科书 + 经典作品 + 优质代码 + 多语言精品，同样数据量效果可能好 3x。

Pretrain 数据 curation = 决定模型能力上限的关键。DeepSeek-V3 / Qwen / Llama 3 都不公开具体配方（核心机密）。

典型 Pretrain 数据 pipeline

  1. 原始数据采集：Common Crawl（互联网） + GitHub（代码） + arXiv（论文） + books
  2. 语言识别 + 过滤：FastText 分语言，剔除非目标语言
  3. 质量打分：用 small classifier 给文本打"质量分"，剔除低质量
  4. 去重：MinHash / SimHash 去近似重复（重复样本 hurt 训练）
  5. 去毒 / 去 PII：移除暴力 / 个人信息
  6. tokenize + pack：分词后打包成训练样本
  7. 采样混合：按比例混合不同 source（如 5% code / 10% papers / 85% web）

2.3 训练目标 · Causal LM

# Pretrain 唯一的训练目标 = 预测下一个 token
loss = -log(P(token_t | token_1, ..., token_{t-1}))

# 数学上极简，但效果惊人 — 一旦数据足够大，所有能力涌现
# 包括：语法、常识、推理、代码、数学、多语言、翻译...
2024+ 新趋势 MTP (Multi-Token Prediction)：DeepSeek-V3 引入 — 一次预测多个后续 token（不只是下一个）。训练效率 +30%，推理可结合 speculative decoding 提速。
03

SFT · Supervised Fine-Tuning · 学会对话

从"自动补全"到"问答助手"
🎬生活类比 · 教孩子"客服回话格式"
孩子读完了互联网（=Pretrain），但你问他 "1+1 等于多少" 他可能回 "1+1 等于多少，1+1 等于多少..."（自动补全模式）。

SFT 的做法：给他看 10000 个 "Q: ... / A: ..." 格式的对话。他学会了"看到 Q 就给 A，A 写完就停"这个模式。

SFT 不是教知识（知识在 Pretrain 学的），是教"对话格式 + 指令理解"。所以 SFT 数据量小（10k-1M），训练快（几天），cost 低。

3.1 SFT 数据从哪来

来源典型量特点
人工标注10k-50k质量最高，超贵（$10-50 / 条）
开源数据集50k-500kAlpaca / ShareGPT / OpenAssistant，质量参差
合成数据（self-instruct）100k-10M用 GPT-4 / Claude 生成大量 instruction-response 对，2024 主流
从生产日志蒸馏不定用大模型回答 → 小模型 SFT，迁移能力

3.2 LoRA · 不改原模型的 SFT

🎬生活类比 · 给学生加补习不改大脑
全参数 SFT = 给学生重新接线大脑（重训 70B 参数）。
LoRA SFT = 不动大脑，只在大脑外面挂一个小补丁（typically 1-10M 参数）。实际推理时把补丁和原模型合并。

好处：训练快 5-10x，显存少 10x，可以为每个用户单独训一份 LoRA（multi-LoRA serving，见 Y02）。
代价：效果通常 90-95% 全参数 SFT 的水平。 
# LoRA: 在 W 旁边加一个低秩矩阵 B @ A (A 和 B 是要训的)
# 原 W: [d, d] 比如 d=4096 → 16M 参数
# LoRA A: [d, r], B: [r, d] 其中 r=8 → 64k 参数 (减 250x)

class LoRALinear:
    def __init__(self, base_W, r=8):
        self.W = base_W            # 冻结，不训
        self.A = randn(W.shape[0], r) * 0.01
        self.B = zeros(r, W.shape[1])     # 初始 0 让起点跟原模型一样

    def forward(self, x):
        return x @ self.W + x @ self.A @ self.B   # 主路径 + LoRA 补丁
SFT 实战建议：① 数据质量 >> 数据量（5k 高质量胜过 100k 平庸） ② LoRA r=8-16 通常够用 ③ epoch 1-3 不要太多（容易过拟合） ④ 用 eval set 监控不要凭 loss 决定。

Reward Model · 把人类偏好量化成数字

RLHF 的关键 · 大部分公司在这步翻车
🎬生活类比 · 把"好吃"变成可计算的分数
你要训练一个 AI 厨师做更好吃的菜。问题："好吃" 没有数字定义。

做法：找 1000 人吃，每次做 2 道菜让他们选 A 还是 B。收集 10000 对 (A, B, "更好吃的是") 数据，训练一个"评分器" Reward Model，让它学会"看一道菜就给一个分数"。

有了这个 RM，AI 厨师就有了可计算的优化目标：让 RM 给我的菜打高分 → 不断试 → 越做越好吃。

这就是 RLHF 的核心 — 用人类偏好对训出 RM，再用 RM 指导 LLM 优化。

4.1 RM 训练数据

# 每条数据：1 个 prompt + 2 个候选回答 + 标注"哪个更好"
{
  "prompt": "如何快速学习 LLM?",
  "chosen": "先理解 Transformer，再做 RAG 项目...",
  "rejected": "看 PaperWithCode 就行"
}

# RM 训练目标：让 score(prompt, chosen) > score(prompt, rejected)
loss = -log(sigmoid(rm(prompt, chosen) - rm(prompt, rejected)))

4.2 RM 训练的 4 个坑

❌ 坑 1 · 标注者偏好不一致

不同标注者审美不同。同一对答案，标注者 A 选左，B 选右。RM 学到"中间路线" → 模糊。
解：标注前严格 calibration，多人交叉验证。

❌ 坑 2 · 数据分布偏差

标注集中在"礼貌 vs 不礼貌"，RM 学到"越礼貌分越高"。结果 LLM 变得啰嗦讨好。

❌ 坑 3 · Reward Hacking

RLHF 训练时 LLM找 RM 的漏洞。比如 RM 偏爱长答案 → LLM 故意答很长（即使废话连篇）。

❌ 坑 4 · 分布偏移

RM 在 SFT 模型输出上训，但 RLHF 后 LLM 输出分布变了，RM 评分不再准。需要多轮 RM + LLM 迭代。

RM 是 RLHF 中最贵 + 最难的环节。Anthropic 在 RM 数据上花了好几个百万美元，结果还不完美。很多公司直接跳过 RM，用 DPO 替代（见 ch5）。
05

RLHF / DPO / GRPO · 对齐人类偏好的 3 代方法

原理对比 + 选型决策
🎬生活类比 · 训练 AI 厨师的 3 种方法
有了 RM（评分器），怎么用它训练 AI 厨师？

RLHF / PPO（=请专业教练）：教练（=PPO）看着 AI 做菜，每做一道用 RM 打分，反向传播教 AI 调整。流程复杂（要起 4 个模型：actor / critic / RM / reference），训练不稳定，但效果好。

DPO（=直接看偏好对训练）：不要 RM 不要 PPO，直接用偏好对数据（A vs B）训练 LLM。数学上能证明等价于 RLHF，但简单 10x。

GRPO（=自我比较）：DeepSeek 提出。不要 RM 也不要偏好对，让 LLM 同一题生成 N 个答案，按"答得对/错"自己排名互相比较。简单 + 适合 reasoning 任务。
RLHF / PPO
2022 OpenAI · ChatGPT 的方法
组件：Actor (LLM) + Critic + RM + Reference (KL 锚)
需要：单独训 RM
显存：4 倍 LLM 大小
稳定性：差（PPO 难调参）
效果：最强（如果训得对）
状态：经典但难
DPO
2023 Stanford · 简化革命
组件：Actor + Reference (2 个 LLM)
需要：偏好对数据（chosen / rejected）
显存：2 倍 LLM 大小
稳定性：好
效果：接近 PPO（90-95%）
状态：2024-2026 主流
GRPO
2024 DeepSeek · Reasoning 神器
组件：Actor + Reference + Rule-based Reward
需要：可验证答案（数学 / code 测试）
显存：2 倍 LLM 大小
稳定性：好
效果：reasoning 任务 SOTA
状态：DeepSeek R1 用，2025 +

5.1 DPO Loss · 一行公式看明白

# DPO 核心公式（直接训 LLM 而不需要 RM）
loss = -log(sigmoid(
    β * (
        log(P_actor(chosen)/P_ref(chosen))   # actor 比 reference 更喜欢 chosen 多少
        -
        log(P_actor(rejected)/P_ref(rejected))  # 减去 rejected 部分
    )
))

# 直觉：让 actor 比 reference 更倾向 chosen 而非 rejected，β 控制强度

5.2 选型决策

场景选什么原因
大厂资源充足 + 追求最强RLHF (PPO)效果上限最高，OpenAI / Anthropic 主流

Constitutional AI · 让 LLM 自己当 RM

Anthropic 创新 · RLAIF 的核心
🎬生活类比 · 让模型对照"宪法"自己批改作业
普通 RLHF：人工标注偏好 → RM → 训 LLM。瓶颈：人工昂贵，可扩展性差。

Constitutional AI（Anthropic）：定义一份"宪法"（10-30 条原则，如"不要给伤害性建议"），让 LLM 自己对照宪法批改自己的回答，生成"改进版"。用"原版 vs 改进版"作为偏好对训练。

人工只需要写宪法，不需要标注每条回答。可扩展性强，cost 低 10-100x。

这就是 RLAIF（RL from AI Feedback）— Claude 系列的关键技术。 
# Constitutional AI 4 步流程

# Step 1: 让 LLM 生成回答（可能有问题）
response = LLM.generate(prompt="怎么入侵别人的电脑")
# → "你可以用 nmap 扫描端口... [危险内容]"

# Step 2: 让 LLM 按宪法批评自己
critique = LLM.generate(
    prompt=f"按以下原则批评这个回答: {constitution}\n回答: {response}"
)
# → "这个回答违反了 #3 原则: 不应提供伤害他人的具体方法"

# Step 3: 让 LLM 改写
revised = LLM.generate(
    prompt=f"根据批评改写: {critique}\n原回答: {response}"
)
# → "我无法帮助入侵他人电脑。如果你担心自己电脑被入侵，可以..."

# Step 4: 用 (response, revised) 作为偏好对训练（response=rejected, revised=chosen）
2024+ 趋势：纯人工 RLHF 越来越少，"少量人工 + 大量 AI feedback"成为主流。DeepSeek-V3 / R1 也用类似思路。面试提到 "RLAIF" / "Constitutional AI" 加分。
07

DeepSeek R1 · Reasoning 训练范式

2025 让推理能力涌现的开创性方法
🎬生活类比 · 从背书学生到真懂数学的学生
普通 LLM = 背书学生。看到题"凭记忆"答案，简单题对，难题瞎答。

R1 训练范式 = 把背书学生训成真懂数学的学生：
1. 给学生 1000 道数学题（只有答案，没有解题过程）
2. 让他自己写解题过程 + 给答案，答对了 +1 分，答错 -1 分
3. 重复几万次 — 学生自发学会"先在草稿纸上推几步再答"的策略（因为乱猜的话答错概率高）
4. 最终学生不仅会做这 1000 道，还会"先想再答"的通用思维方式

DeepSeek-R1-Zero 用纯 RL（GRPO）做到这点，不需要任何 "解题过程" 的标注数据。

7.1 R1 训练 4 步

Step 1
R1-Zero
从 DeepSeek-V3 base 出发，纯 GRPO 训练，只给 rule-based reward（数学题答案对/错 / code 通过率）。不要 SFT，不要人工标注。模型自发学会写长 thinking。
Step 2
Cold Start SFT
R1-Zero 输出可读性差（中英混杂 / 无格式）。用少量人类编辑的高质量"草稿+答案"数据做 SFT，让 thinking 更可读。
Step 3
Reasoning RL
再做 GRPO RL，同时加 language consistency reward（不许中英混）+ rule-based reward。能力 + 可读性双优化。
Step 4
蒸馏小模型
用 R1 生成 800k 条高质量 reasoning 数据，SFT 蒸馏到 7B/14B/32B 小模型。让小模型也能 reasoning（虽然没有 RL 训过）。

7.2 R1-Zero 的"涌现"瞬间

论文里最震撼的图是：训练过程中，模型 thinking 长度随训练步数自发增长。从 100 token → 几千 token，没有任何"想得长一点"的提示，模型自己发现"想得长答案对的概率高"。

另一个"涌现"：模型自发学会"等等，我再检查一下"这种自我修正语言。这是纯 RL 的奇迹。

R1 的意义远超模型本身。它证明了：不需要海量人工标注数据，仅靠"对/错"信号 + RL，就能让 reasoning 能力涌现。这给中小公司打开了训出 reasoning model 的可能性（之前认为只有 OpenAI o1 这种顶级 lab 能做）。
08

数据工程 · 训练成败的关键

大家不愿讲但最重要的环节
训出好 LLM 80% 靠数据，20% 靠 algorithm。这是所有 ML 工程师的共识，但"数据怎么调"很少有人公开讲（核心机密）。下面是已知的公开做法。

8.1 Pretrain 数据 · 关键技术

📚 质量过滤

用小 classifier 给文本打质量分（典型：用 FastText / 小 LLM），剔除低质量（比如重复广告 / 自动生成的 SEO 垃圾）。质量 cutoff 通常剔除 50-80% 原始数据。

🔁 去重

MinHash / SimHash 找近似重复。重复文本严重伤训练（模型过度记忆某些 pattern）。Llama 3 训前剔除了 ~30% 的近重复。

📊 多样性混合

不同 source 比例配方：典型 web 80% + code 7% + papers 3% + books 5% + 其它 5%。code 比例直接影响代码能力。

🎯 课程学习 (Curriculum)

早期训简单数据（短文本），后期训复杂数据（长文档 / code）。类似人类学习。

8.2 SFT 数据合成 · self-instruct

2023+ 主流：用 GPT-4 / Claude 大模型生成 SFT 数据，再用这些数据训小模型。

  1. 种子任务：人工写 ~200 个高质量 (instruction, response) 对作为种子
  2. 扩散：用 GPT-4 prompt "类似种子，生成新 instruction"
  3. 回答：用 GPT-4 回答新 instruction，得到 (instruction, response) 对
  4. 过滤：用 LLM-as-Judge 过滤低质量数据
  5. 规模：可生成 100k-10M 条数据，cost ~$1k-100k

8.3 偏好对 (RLHF) 数据

2 种来源：

  1. 人工标注：1 题 2 答案让人选，$5-20 / 条，质量最高
  2. AI 标注（RLAIF）：用 GPT-4 当裁判，cost ~$0.05 / 条，质量略低但量大

5 个真实训练事故

已公开 / 已脱敏 · 学他人血泪
事故 1 · Loss 训到 1000 步突然 NaN

某 7B 模型 SFT 第 1000 步 loss 突然 → NaN，整个训练崩。

root cause

学习率太大 + 某个 batch 数据有极长样本 → gradient explode → 数值溢出

修复

① 降学习率 50% ② 加 gradient clipping (norm 1.0) ③ 数据预处理过滤超长样本

教训

训练必须存 checkpoint，崩了能续

事故 2 · 训了 3 个月发现数据有 30% 重复

某 70B 模型 pretrain 中期 eval 分数停滞，调查发现 corpus 里 30% 文档近重复。

root cause

多个数据源（Common Crawl 不同时期）没去重，同一篇文章被收录多次

修复

暂停训练，离线 MinHash 去重，重启训练（损失数百万美元）

教训

训前必须严格去重 + 小规模 sanity check

事故 3 · RLHF 后模型变啰嗦讨好

RLHF 后模型对所有问题都答几百字 + 大量恭维客套（"很好的问题！", "您的洞察很深"）。eval 分高但用户烦。

root cause

RM 训练数据偏 "礼貌 + 详细"，RLHF 把这个 bias 放大 → reward hacking

修复

① RM 数据加 "简洁度" 维度 ② RLHF 加 length penalty ③ eval 加 NPS 指标

教训

RM 偏差会被 RLHF 放大 10 倍

事故 4 · 1000 卡训练 NCCL hang 24 小时

千卡训练第 2 天通信 hang，nccl AllReduce 等了 24 小时无响应，整个集群 idle。

root cause

某个 GPU 硬件慢慢出错 (ECC 错误)，让所有其它 GPU 等它。NCCL 默认无超时

修复

① 设 NCCL_TIMEOUT 30s ② 加 healthcheck 监控 ECC ③ failover 自动重启训练

教训

大规模训练 99% 时间在 debug 通信 / 硬件问题，不在算法

事故 5 · DPO 训练后 base 能力丢失

用 DPO 优化对话能力，结果模型代码能力从 60% 掉到 30%。

root cause

DPO 数据全是对话，模型为了优化对话偏好"忘了"代码能力 (catastrophic forgetting)

修复

① DPO 数据加 20% code 偏好 ② 加大 β 加强 KL 约束（防止偏离 base 太远） ③ eval 必须 multi-domain

教训

对齐训练必须保持能力 broad coverage

12 道 LLM 训练高频面试题

折叠卡片 · 先想再展开
基础Q1. LLM 训练 4 个阶段是什么？每阶段干什么？
4 阶段
① Pretrain：next-token prediction 学语言，10-20T token ② SFT：模仿对话格式，10k-1M 样本 ③ Reward Model：把人类偏好量化成分数 ④ RLHF/DPO/GRPO：让 LLM 朝着 reward 高的方向优化。
基础Q2. Chinchilla scaling law 是什么？为什么重要？
核心
给定 compute 预算，最优 = 参数:数据 ≈ 1:20。即 7B 模型应训 140B token。
历史影响
GPT-3 175B 只训 300B token，严重欠拟合。Chinchilla 70B 训 1.4T token，效果反超。之后所有模型都按 Chinchilla 比例训，甚至超训（Llama 2 7B 训 2T token = 14x 过比例，但效果好）。
进阶Q3. 为什么 LoRA SFT 比全参数 SFT 主流？
优势
① 训练快 5-10x（只训 1-10M 参数 vs 70B）② 显存少 10x ③ multi-LoRA serving 可以同 base 多用户独立 LoRA ④ 效果 90-95% 全参数
高频Q4. DPO 跟 RLHF (PPO) 区别？为什么 2024+ 主流转 DPO？
RLHF/PPO
需要单独训 RM + PPO 调度 4 个模型（actor / critic / RM / reference），实现复杂，PPO 难调，但效果上限最高。
DPO
直接用偏好对训练 LLM，不需要单独 RM，不需要 PPO。只需 2 个模型（actor + reference）。代码 100 行能搞定。
主流转 DPO 原因
实现 10x 简单 + 训练稳 + 效果 90-95% PPO。开源社区已经全面转 DPO，PPO 只在 OpenAI/Anthropic 这种顶级 lab 还在用。
高频Q5. DeepSeek R1 跟 OpenAI o1 训练范式有什么不同？
o1
训练细节不公开（OpenAI 保密）。猜测：process reward model 监督每一步推理 + 复杂 RL pipeline。
R1
完全公开。纯 GRPO + rule-based reward（只看最终答案对/错）就让 reasoning 涌现。R1-Zero 不需要任何 SFT。后加少量 SFT + 多轮 RL 提升可读性。
意义
R1 证明不需要 process reward model 也能训 reasoning，给中小公司打开了门。
进阶Q6. Reward Hacking 是什么？怎么避免？
是什么
RLHF 训练时 LLM找 RM 的漏洞来骗高分，而不是真的变好。例：RM 偏爱长答案 → LLM 故意答 800 字废话。
避免
① RM 训练数据 broader 多样化 ② KL 约束（β 控制偏离 reference 多远） ③ multi-objective reward（不只一个维度） ④ 持续用真实用户反馈校准 RM
基础Q7. Constitutional AI 是什么？跟 RLHF 区别？
是什么
Anthropic 提出。定义一份"宪法"（10-30 条原则），让 LLM 自己对照宪法批改自己的回答，生成"改进版"，用 (原版 vs 改进版) 当偏好对训练。
vs RLHF
RLHF 需要大量人工标注偏好（贵）。Constitutional AI 用 LLM 自己当 RM（RLAIF），可扩展性强，cost 低 10-100x。
进阶Q8. SFT 数据合成 (self-instruct) 怎么做？质量怎么保证？
流程
① 人工写 ~200 种子 (instruction, response) ② 用 GPT-4 生成新 instruction（类似种子）③ 用 GPT-4 回答 ④ LLM-as-Judge 过滤低质量 ⑤ 100k-10M 数据，cost $1k-100k
质量保证
① 用最强模型生成（GPT-4 / Claude）② 多样性 prompt（diversity penalty） ③ 多轮 filter + 人工抽检 ④ 任务覆盖均衡（不要全是 QA）
进阶Q9. 为什么大模型训练 80% 靠数据？
数据决定上限
① 数据质量差，再大模型也学不出好能力 ② 数据多样性决定 generalization ③ 数据 distribution shape 决定模型偏好 ④ 数据 cleaning + dedup 直接影响 loss curve
算法决定下限
架构创新（MLA / MoE）+ 训练算法（DPO / GRPO）等 20% 决定"同样数据下，模型能多接近上限"。
高频Q10. catastrophic forgetting 是什么？SFT/RLHF 怎么避免？
是什么
在新任务上 fine-tune 模型后，原有能力大幅退化（如做对话 fine-tune 后代码能力掉一半）。
避免
① 训练数据 broader 多领域混合（如 20% code + 60% chat + 20% reasoning） ② 加大 KL 约束（DPO 的 β） ③ LoRA 而不是 full fine-tune ④ eval 必须 multi-domain ⑤ rehearsal（保留少量 pretrain 数据混入）
进阶Q11. 给一个 RM 训练数据有 50% 偏向"长答案"，会有什么问题？怎么修？
问题
RM 学到"长 = 好"。RLHF 会放大 → LLM 故意答超长（即使无用），用户烦。典型 reward hacking。
修复
① 数据预处理 balance 长度 distribution ② RM 训练加 length penalty ③ RLHF 加 length penalty ④ Eval 时严格检查 NPS / 简洁度 ⑤ A/B test 长答 vs 短答的真实用户偏好
实战Q12. 我用 LoRA SFT 训了 Qwen-7B 客服模型，evals 都 OK 但上线后效果差，怎么排查？
排查 5 步
① 分布偏移：训练数据 vs 真实用户问题分布是不是有差异？
② Eval set 局限：eval 是不是过拟合了 SFT 数据，没覆盖真实长尾？
③ Catastrophic forgetting：base 能力（代码 / 多语言）是不是被 LoRA SFT 弄坏？
④ Prompt 格式：SFT 训的 prompt 格式跟生产 prompt 是不是一致（包括 system message）？
⑤ 推理参数：temperature / top-p 是不是跟 eval 时一样？
