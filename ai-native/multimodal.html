01

视觉理解 · CLIP / ViT · 让模型看懂图

现代多模态的地基
🎬生活类比 · 教小孩认猫
想象教小孩认"猫"。
① 你给他看 1000 张图片 + 配文字"这是猫"
② 看多了他形成"猫的概念" — 看图能立刻说出"猫"

CLIP 就是这么训的：从互联网搜 4 亿张 (图片, 描述) 对，用 contrastive learning 让模型学会"图跟文字对得上"。训完后图片和文字都被映射到同一个向量空间 — 语义相近的图和文字向量距离近。

1.1 ViT · Vision Transformer

ViT 把图片切成 16×16 patches，每个 patch flatten 成 token，跟文本 token 一样喂给 Transformer。这是统一架构的关键。

# ViT 流程
image = load("cat.jpg")              # 224x224x3
patches = split_to_patches(image, 16)  # 14×14 = 196 patches
patch_embeds = linear(patches.flatten())  # 每 patch → token
patch_embeds = patch_embeds + position_embed
output = transformer(patch_embeds)        # 跟文本一样的 attention

1.2 CLIP · 对比学习的革命

CLIP 的核心：训练时同 batch 内 N 张图 × N 个描述，正确对（图 i 跟描述 i）相似度最大，错误对最小。本质是"把图和文字拉到同一个语义空间"。

1.3 应用

🔍 零样本图像分类

给图片 + N 个候选类别文字 → 算图跟每个文字的相似度 → 最像的就是分类。不需要为每个新类别重新训。

🎨 图文检索

用文字搜图（"穿红衣的女孩骑自行车"）或反向。电商图搜核心。

🤖 多模态 LLM 的基础

CLIP image encoder 是 LLaVA / Qwen-VL 等 VLM 的标准组件之一。

🎬 Diffusion 的引导

CLIP 给 text-to-image 模型（DALL-E / Stable Diffusion）当"评判器"。

02

VLM · Vision-Language Model

让 LLM 能看图说话
🎬生活类比 · 给 LLM 装眼睛
普通 LLM = 只能读文字的人。
VLM = 这个人装上一双能"翻译"图像的眼睛（vision encoder），把看到的图翻译成 LLM 能懂的 token 喂进去。然后 LLM 像处理文字一样处理这些"图像 token"，输出文字。

2.1 标准架构

  1. Vision Encoder（如 ViT / SigLIP）：图 → 视觉 features
  2. Projector / Adapter（如 MLP / Q-Former）：视觉 features → LLM token 空间
  3. LLM Decoder（如 Qwen / Llama）：把"图像 token"跟文字 token 拼在一起做 next-token prediction

2.2 主流 VLM 对比（2026）

模型开源特点
Qwen2.5-VL✅中文 OCR + 长视频理解 · 国内 SOTA
InternVL 2.5✅上海 AI Lab · 通用能力强
LLaVA-OneVision✅学术界经典 · 训练范式公开
GPT-4o / Claude 3.5 Sonnet❌商业 SOTA · 综合能力顶级
Gemini 2.0❌原生多模态 · 视频长上下文优势

2.3 VLM 的局限

⚠ 计数 / 空间推理

"图中有几个苹果" — 当下 VLM 经常错 1-2 个。OCR 数字识别准但概念性计数差。

⚠ 高分辨率细节

默认 224×224 输入丢细节。需 dynamic resolution（如 Qwen-VL）。

⚠ 视频时序

长视频每帧独立看，缺时间连贯性。最新模型加 temporal attention 改善。

⚠ 幻觉更严重

看不清的部分容易瞎猜（"图里有什么" → 编造对象）。

03

语音 · Whisper / TTS

让 AI 听说

3.1 ASR (语音识别) · Whisper

OpenAI Whisper（2022）是 ASR 的革命：在 68 万小时多语言数据上训练，对噪声 / 口音 / 多语言极鲁棒。开源后席卷工业界。

  • 架构：Encoder-Decoder Transformer。音频 → mel-spectrogram → encoder → decoder 输出文字 token
  • 多任务：transcribe (识别) / translate (翻译到英文) / detect language / timestamp
  • 大小：tiny / base / small / medium / large（39M - 1.5B 参数）
  • 2026：Whisper Large v3 是开源 ASR 主流

3.2 TTS · 文字转语音

方案特点
传统 (Tacotron / FastSpeech)2 阶段：text → mel-spec → vocoder → wave。质量好但慢
VALL-E / NaturalSpeech用 LLM 风格生成 audio token，能 zero-shot 克隆声音（3 秒样本就够）
CosyVoice / GPT-SoVITS开源 zero-shot 克隆，中文 SOTA
ElevenLabs / OpenAI TTS商业 SOTA
2026 趋势：统一的 "speech LLM"。GPT-4o realtime / Qwen2-Audio / Gemini Live 是原生 audio token 处理 — 不再 ASR → LLM → TTS 三段式，延迟从 1-2s 降到 200ms，能真正自然对话。
04

Diffusion 生成模型

Stable Diffusion / DALL-E / Flux 的核心
🎬生活类比 · 从沙子里"擦"出图
想象你想画一只猫。
Diffusion 不是"从空白开始画"，而是从随机噪声开始（一张完全的"雪花屏"），一步步把噪声"擦掉"，最后浮现出猫。

训练时：拿真实图片加 T 步噪声变成纯噪声，让模型学"反向去噪"。推理时：从纯噪声开始，按训过的反向步骤一步步去噪。

4.1 主流架构演进

  • DDPM（2020）：原始 diffusion，1000 步 denoise，慢
  • Stable Diffusion（2022）：在 latent space 做 diffusion，速度快 10x
  • DiT (Diffusion Transformer)（2023）：用 transformer 替代 UNet，Sora 用
  • Flow Matching / Rectified Flow（2024+）：数学更简洁，Stable Diffusion 3 / Flux 用

🎮 交互演示 · Diffusion 去噪 step-by-step

从最右 (T=1000, 纯噪声) 一步步去噪到最左 (T=0, 清晰图)。拖滑块或自动播放看 16×16 像素 grid 怎么逐步"浮现"出目标（字母 "A"）。

拖滑块从 T=1000 (纯噪声) 到 T=0 (清晰图)，每步 model 预测"应该去掉多少噪声"

4.2 加速

Diffusion 慢的核心是"多步采样"（典型 20-50 步）。加速方法：

  • LCM (Latent Consistency)：训练时学"几步就能 denoise 完"，把 50 步降到 4 步
  • DMD (Distribution Matching Distillation)：1 步生成（蒸馏自多步模型）
  • FLUX.1 schnell：4 步生成接近 50 步质量

05

Sora · 视频生成

2024 开启 video diffusion 时代

5.1 Sora 关键创新

  1. Spacetime Patch：把视频切成 3D patch（时间 × 高 × 宽），每个 patch 是一个 token
  2. DiT 架构：用 transformer 而非 UNet，可 scale up
  3. 世界模型 emergent：模型自发学到物理规律（重力、流体）— 至少表面上
  4. 视频长度：单次生成 60s（突破之前 4-8s）

5.2 2026 视频生成生态

模型特点
OpenAI Sora第一个突破性，长视频质量最强
Veo 2 / 3 (Google)电影级质量
Runway Gen-3商业最成熟，创作者用
HunyuanVideo（腾讯）开源 SOTA
Kling AI（快手）国内商业 SOTA
Wan 2.1（阿里）开源，中文场景
视频生成的 cost：1 秒视频 ~$0.1-1.0，比 1k token LLM 贵 1000x。所以 2026 主流不是"实时生成"，是"创作工具" — 用户描述+生成+精修。
06

智驾多模态融合

VLM / VLA 上车
🎬生活类比 · AI 司机的眼+耳+脑
智驾系统 = 多模态融合的极致案例：
① 多摄像头（前/后/左/右/环视）= 视觉
② 毫米波雷达 + 激光雷达 = 距离感
③ IMU + GPS = 自身位置/姿态
④ 地图 + 导航 = 路网知识
⑤ 用户语音指令（"开慢点"）= 语言

AI 系统要把所有这些融合成一个"场景理解"，输出方向盘转角和油门。

6.1 端到端 vs 模块化

🧱 传统模块化

  • 感知 (detect car/pedestrian)
  • 预测 (trajectory)
  • 规划 (plan path)
  • 控制 (steer/throttle)

优势：可解释，可调试。劣势：模块边界 hard-code 限制。

🧠 端到端 (E2E)

  • 多传感器 → 大 VLM → 直接输出 action
  • Tesla FSD v12 / 小鹏 XNGP 2025+ 用

优势：emergent intelligence。劣势：黑盒，难 debug，安全验证难。

6.2 VLA · Vision-Language-Action

从单一视觉 → action（VA）扩展到 视觉+语言 → action（VLA）。能听人话"在前方红绿灯左转"或"开慢点跟着前车"。具身机器人也用同范式。

07

10 道多模态面试题

基础Q1. CLIP 怎么训练？为什么 zero-shot 分类能 work？
训练
contrastive learning：4 亿对 (图, 文)，同 batch 内正确对相似度最大错误对最小。图和文字被映射到同一向量空间。
zero-shot
新类别时不需要重新训，只需把类别名（"猫"/"狗"）转 embedding，跟图 embedding 算相似度，最像的就是分类。
基础Q2. ViT 跟 ResNet/CNN 比有什么优势劣势？
优势
① 统一架构（跟 LLM 一样 transformer）② 全局感受野 ③ scale up 效果更好
劣势
① 需要更多数据训（CNN 有 inductive bias）② 高分辨率计算贵（patch 数平方）
进阶Q3. VLM 的 architecture 主流 2 种？区别？
2 种
① MLP projector（LLaVA 风）：vision encoder + 简单 MLP + LLM。简单但 token 数多
② Q-Former（BLIP-2 风）：用一组 learned query token 压缩视觉信息。token 少但复杂
2026 主流
MLP 简单胜出（Qwen-VL / InternVL 都用）。Q-Former 慢慢淡出。
进阶Q4. VLM 4 个常见局限？
4 局限
① 计数/空间推理差 ② 高分辨率细节丢 ③ 视频时序弱 ④ 幻觉更严重（看不清就编造）
基础Q5. Whisper 为什么这么鲁棒（噪声/口音/多语言）？
数据
68 万小时多语言数据训练，涵盖各种噪声/口音/方言。规模碾压学术界 ASR 模型 100x。
进阶Q6. Diffusion 模型为什么能从噪声生成图像？数学直觉？
训练
给真图加 T 步噪声 → 纯噪声。让模型学"反向去噪"（每一步预测要去掉的噪声）。
推理
从纯噪声开始，反向去噪 T 步，逐渐浮现出图像。数学上 = 学 reverse diffusion process 的 score function。
进阶Q7. LCM / DMD 怎么把 diffusion 从 50 步加速到 1-4 步？
原理
用多步模型蒸馏到少步模型。LCM 训练时学"我跳几步预测就能跟原模型 N 步等效"。DMD 通过分布匹配 1 步生成。
高频Q8. Sora 跟之前视频生成有什么不同？
3 创新
① Spacetime Patch：视频切 3D patch 当 token ② DiT 架构：transformer 替代 UNet，可 scale ③ 长视频：60s 突破之前 4-8s
emergent
大规模训练后自发学到物理规律（重力/流体），虽然不完美但显著优于上一代。
进阶Q9. 智驾端到端 (E2E) 比模块化方案的优劣？
优势
① emergent intelligence（学到模块化设计想不到的策略）② 不被模块边界 hard-code 限制
劣势
① 黑盒，难 debug ② 安全验证难（ISO 26262 难认证）③ 失败模式不可预测
高频Q10. 2026 多模态主流趋势是什么？
3 趋势
① 统一 token：图/文/音/视频全部 token 化用同一 transformer（GPT-4o, Gemini 2.0）② 实时对话：原生 audio LLM 让 latency 从 1-2s → 200ms ③ VLA 具身：视觉+语言+动作统一，机器人和车端都用
