参数高效微调 PEFT：LoRA / QLoRA 及其家族

给大模型做定制，早年要全量微调（更新全部几十上百亿参数），显存和存储成本高到普通团队玩不起。PEFT（参数高效微调）换个思路：冻住原模型，只训练极少量新增参数，就能达到接近全量微调的效果，显存需求砍到几分之一甚至几十分之一。本篇从 LoRA 的低秩分解原理讲起，到 QLoRA 让单张消费级显卡微调 65B 模型，再到 DoRA / Adapter / Prefix / Prompt Tuning 家族与选型，配一个可交互的 LoRA 参数量/显存计算器。

为什么不全量微调

显存、存储、多任务——全量微调的三座大山

全量微调（Full Fine-Tuning）就是把预训练模型的所有参数都拿来继续训练。效果好，但代价惊人：训一个 7B 模型，光是参数、梯度、优化器状态（Adam 要存一阶+二阶动量）就要占十几倍于模型本身的显存；而且每个任务都要存一份完整的微调后模型（7B ≈ 14GB）。

全量微调像为了改一句台词，把整部电影重拍一遍——演员、场景、后期全部重来，又贵又慢，还得为每个版本存一整部片子。PEFT 像只补拍那一个镜头，然后做成一个"补丁"叠加到原片上：原片不动，补丁只有几 MB，想要哪个版本就叠哪个补丁。同一部原片 + 一堆小补丁，就能有无数定制版本。
成本项全量微调PEFT（如 LoRA）
训练显存极高（含全部梯度+优化器状态）大幅降低（只优化少量参数）
每任务存储一整份模型（GB 级）一个适配器（MB 级）
多任务部署每任务一份权重共享主干 + 热插拔适配器
灾难性遗忘易改动主干、遗忘通用能力主干冻结，通用能力保留更好
效果上限最高多数任务接近全量
PEFT（Parameter-Efficient Fine-Tuning，参数高效微调）：一类"冻结预训练模型主体、只训练少量（新增或选定）参数"的微调方法总称。目标：用远小于全量微调的显存/存储/算力，达到接近全量的下游效果。LoRA 是其中最流行的一种，QLoRA、Adapter、Prefix/Prompt Tuning 等都属此家族。
01

PEFT 全景：冻主干，只动一小撮

所有 PEFT 方法的共同套路

别看方法名花样多，PEFT 的核心思想只有一句：把预训练权重冻住不动，只在旁边加/选一小撮可训练参数去"拟合"新任务。区别只在于"这一小撮参数加在哪、长什么样"。

四类思路：加旁路低秩矩阵（LoRA 系）、层间插小模块（Adapter）、训软提示（Prefix/Prompt）、微调极少已有参数（BitFit/IA³）。LoRA 系是当下绝对主流。
02

LoRA：用两个「瘦矩阵」逼近权重更新

Low-Rank Adaptation · PEFT 的事实标准

2.1 低秩分解：核心就一个式子

LoRA 的洞察：微调时权重的变化量 ΔW，其实"信息含量"很低——它可以用两个又瘦又小的矩阵相乘来近似。于是不去改巨大的 W，而是在旁边并联一条 ΔW = B·A，只训练 A、B 这两个小矩阵。

LoRA（Low-Rank Adaptation）：冻结原权重 W（形状 d×d），在旁边加一条低秩旁路：h = Wx + (B·A)x，其中 A 是 r×d、B 是 d×r，秩 r ≪ d（如 r=8，d=4096）。只训练 A、B。可训练参数从 d×d 骤降到 2×d×r。推理时可把 B·A 合并回 W，零额外延迟。
A 通常用高斯初始化、B 初始化为 0（保证训练开始时旁路输出为 0，不破坏原模型）。缩放因子 α/r 控制旁路的影响强度。
秩 r 与 α（alpha）：r 是旁路矩阵的秩（瓶颈宽度），越大容量越强但参数越多，常用 8/16/32/64。α 是缩放超参，实际旁路按 α/r 缩放；经验上常设 α=2r 左右。r 太小欠拟合、太大接近全量失去意义。

03

QLoRA：4-bit 量化主干 + LoRA

让单张 24GB 消费卡也能微调 65B 模型

LoRA 省了"要训练的参数"，但冻结的主干本身仍要加载进显存——65B 模型光是 FP16 权重就要 130GB，单卡放不下。QLoRA 的杀手锏：把冻结的主干量化到 4-bit 存放（省 4 倍显存），再在其上做 LoRA。主干反正不训练，量化损失可接受；LoRA 旁路仍用高精度训练。

主干是本"只读的大词典"（65B），你查它但从不改它。QLoRA 说：既然只读，何必用最清晰的精装大部头占满书桌？换成压缩版微缩胶片（4-bit）放着，查阅够用就行；你真正要写的批注（LoRA）另用高清好纸写。于是原本要占满整间屋子的词典，压进一个抽屉，桌上就腾出地方给你干活了。
QLoRA：在 LoRA 基础上，把冻结主干量化到 4-bit 存储与前向计算，从而把显存需求再砍数倍。三项关键技术：NF4（4-bit NormalFloat）——为正态分布权重设计的 4-bit 数据类型，比普通 int4 更保精度；Double Quantization（双重量化）——连量化用的缩放常数也量化，再省一点；Paged Optimizer——用统一内存把优化器状态在显存不足时换出到内存，防 OOM。QLoRA 让 65B 模型可在单张 48GB（甚至 24GB）显卡上微调。
QLoRA 把冻结主干压到 4-bit（约 1/4 显存），LoRA 旁路仍高精度训练。以极小的精度代价换来"消费级硬件微调大模型"的能力。
QLoRA 是"低成本微调"的默认选择：效果接近 16-bit LoRA，显存却省数倍。代价是量化/反量化带来一定训练速度下降，且推理时若要合并权重需注意精度处理。追求极致效果/速度时才回到 16-bit LoRA 或全量。
04

其它 PEFT 方法：家族一览

知道它们存在、各自的定位即可

DoRA（Weight-Decomposed LoRA）

把权重分解成"幅度 + 方向"，方向部分用 LoRA 更新。在低秩下比原始 LoRA 更接近全量微调效果，是 LoRA 的进阶改良。

Adapter

在 Transformer 层间插入小"瓶颈"模块（降维→非线性→升维），只训这些模块。PEFT 的开山思路之一，但推理时有额外层、带延迟（不像 LoRA 能合并）。

Prefix / Prompt Tuning

不改权重，只训练一段可学习的"软提示"向量拼在输入前。参数最省，但表达力有限、对超参敏感，效果通常不如 LoRA。

BitFit / IA³

BitFit：只训练模型里的 bias 项，极致省参。IA³：学习几组缩放向量去调制激活。都属"极简 PEFT"，特定场景下够用。

这些方法多数是"学术谱系"，工程上LoRA / QLoRA 是压倒性主流（生态、工具、社区适配器最全）。DoRA 作为 LoRA 的改良在追求效果时值得一试；Prefix/Adapter 更多是理解 PEFT 演化史的知识点。

对比与选型

一张表 + 一句决策

方法可训练参数显存推理延迟效果定位
全量微调100%极高无额外上限最高算力充足、追极致
LoRA<1%中（主干 FP16）可合并=0接近全量主流首选
QLoRA<1%低（主干 4-bit）可合并接近 LoRA显存紧张首选
DoRA~LoRA中略增优于 LoRA追效果的改良
Adapter小中有额外层中上历史/特定场景
Prefix/Prompt极小低略增中极省参、任务简单
一句话选型：显存够 → LoRA；显存紧 → QLoRA；已用 LoRA 想再榨效果 → 试 DoRA。绝大多数团队在 LoRA/QLoRA 之间二选一就够了，别过度纠结冷门方法。
PEFT 不是万灵药：当任务需要模型学习大量新知识（而非新"风格/格式"）时，LoRA 可能不够，需更大的 r 甚至全量微调。经验法则：教模型"怎么说话/什么格式"，LoRA 很行；灌全新领域知识，优先考虑 RAG 或全量/继续预训练。
06

落地：合并权重 / 多适配器 / serving

训完之后怎么用起来

合并 vs 不合并

推理时可把 B·A 合并回 W（merge_and_unload），得到一个普通模型、零额外延迟；也可不合并、运行时动态叠加，方便热切换不同适配器。

多适配器热插拔

同一主干 + 多个 LoRA 适配器（每个几十 MB），按请求动态加载不同适配器 → 一份显存服务多个"定制模型"。vLLM 等已支持 multi-LoRA serving。

工具栈

HuggingFace PEFT 库（LoRA/QLoRA/DoRA…一站式）+ bitsandbytes（4-bit 量化）+ TRL（SFT/DPO 训练）。是事实标准组合。

常见坑

注入层选错（一般注 q/k/v/o + MLP 投影收益好）、r/α 不匹配、QLoRA 合并时精度处理不当、adapter 与 base 版本对不上。上线前务必 eval。

LoRA 的"适配器即插件"特性，让它天然适合多租户/多任务产品：一个基座模型 + 每个客户/任务一个小适配器，成本与运维远优于"每个都存一份大模型"。这也是 PEFT 在工程上最大的红利之一。

小结 · 面试 · 术语表

收束成判断力

一页纸总结

① 冻主干只训一小撮

PEFT 用远小于全量的成本达到接近效果，还避免灾难性遗忘、适配器可热插拔。

② LoRA=低秩旁路

ΔW≈B·A，秩 r≪d，只训 A/B，参数砍到千分之一；推理可合并、零延迟。

③ QLoRA=4bit主干+LoRA

NF4+双重量化+paged optimizer，单卡微调 65B。显存紧张的默认选择。

面试自测（点击展开）

基础为什么要 PEFT？相比全量微调好在哪？
动因全量微调显存/存储/多任务成本极高（梯度+优化器状态十几倍模型大小；每任务存一整份权重）。
PEFT冻主干只训少量参数：显存大降、每任务只存 MB 级适配器、可热插拔、少遗忘、效果接近全量。
核心LoRA 的原理是什么？r 和 α 怎么理解？
原理冻结 W，旁加低秩 ΔW=B·A（r≪d），只训 A、B。h=Wx+B·A·x。参数从 d² 降到 2dr。
r旁路秩=容量瓶颈，常用 8/16/32；太小欠拟合、太大失去意义。
α缩放超参，旁路按 α/r 缩放，常设 α≈2r。B 初始化为 0 保证起始不扰动原模型。
推理可 merge 回 W，零额外延迟——这是它优于 Adapter 的关键。
辨析QLoRA 在 LoRA 上加了什么？为什么能省这么多显存？
关键把冻结主干量化到 4-bit 存储/前向（主干反正不训，量化损失可接受），LoRA 旁路仍高精度。
三件套NF4（适配正态分布的 4-bit 类型）、Double Quantization（连缩放常数也量化）、Paged Optimizer（防 OOM）。
效果65B 可在单张 48GB 卡微调，效果接近 16-bit LoRA，代价是训练略慢。
工程什么任务 LoRA 就够，什么时候得全量微调或改用 RAG？
LoRA 够教模型"风格/格式/语气/遵循指令"这类行为塑形——低秩足以拟合。
要更多需注入大量新知识时，LoRA 容量可能不足：加大 r、DoRA，或全量/继续预训练。
改 RAG若知识频繁更新/需可溯源，别微调，用 RAG 更合适——微调记死知识，RAG 动态检索。

术语表速查 · Glossary

① 总览

PEFT Parameter-Efficient Fine-Tuning冻主干、只训少量参数的微调方法总称。低成本达到接近全量效果。
全量微调 Full Fine-Tuning更新模型全部参数。效果上限高但显存/存储/多任务成本极大。
灾难性遗忘 Catastrophic Forgetting微调改动主干导致原有通用能力退化。PEFT 冻结主干可缓解。

② LoRA 系

LoRA低秩旁路 ΔW=B·A（r≪d），只训 A/B，可合并回 W 零延迟。PEFT 事实标准。
秩 r / αr=旁路秩(容量)，常 8~64；α=缩放超参，旁路按 α/r 缩放，常 α≈2r。
QLoRA4-bit 量化冻结主干 + LoRA。含 NF4、双重量化、paged optimizer。单卡微调大模型。
NF4 4-bit NormalFloat为正态分布权重设计的 4-bit 数据类型，比普通 int4 更保精度。QLoRA 核心。
DoRA权重分解为幅度+方向，方向用 LoRA。低秩下比原 LoRA 更接近全量效果。

③ 其它与落地

Adapter层间插小瓶颈模块，只训该模块。推理有额外层带延迟（不能像 LoRA 合并）。
Prefix / Prompt Tuning训练可学习的软提示向量拼在输入前，不改权重。最省参但表达力有限。
BitFit / IA³极简 PEFT：只训 bias（BitFit）/ 学缩放向量调制激活（IA³）。
multi-LoRA serving一份主干 + 多个适配器热插拔，按请求切换"定制模型"。vLLM 等支持。
