排序家族 ① — sort 内部 introsort

用法 · 严格弱序的陷阱（高频面试题）

排序⭐ 必考

std::sort — Introsort 内省排序详解

std::sort 用什么算法?

不是单一算法 — 是 introsort（内省排序）：3 个算法的混合体。

  1. 开始：quicksort（分治 + pivot）
  2. 递归深度 > 2 * log2(N)：切到 heapsort（防最坏 O(n²)）
  3. 分段 < 16 元素：用 insertion sort（小数组缓存友好）

结果：最坏 O(n log n)，常数因子小，几乎是「最优通用排序」。

关键约束

  • 需要随机访问迭代器（vector/array/deque/string）
  • 不稳定（等价元素的相对顺序可能改变）
  • 比较器必须满足严格弱序（strict weak ordering）—— 违反就是 UB

用法

std::vector<int> v{3,1,4,1,5,9,2,6};
std::sort(v.begin(), v.end());                          // 默认升序
std::sort(v.begin(), v.end(), std::greater<>{});           // 降序

std::vector<User> users;
std::sort(users.begin(), users.end(),
    [](const User& a, const User& b){ return a.score > b.score; });

// C++20 ranges 风格 —— 推荐
std::ranges::sort(v);
std::ranges::sort(users, std::greater<>{}, &User::score);   // 投影!

严格弱序的陷阱（高频面试题）

// ❌ 错误：浮点 NaN 破坏严格弱序，UB
std::vector<double> v{1.0, 2.0, std::nan(""), 3.0};
std::sort(v.begin(), v.end());                   // UB! NaN 与任何数比较都返回 false

// ❌ 错误：<= 不是严格弱序
std::sort(v.begin(), v.end(), [](int a, int b){ return a <= b; });   // UB

// ✅ 正确：必须是 <
std::sort(v.begin(), v.end(), [](int a, int b){ return a < b; });
严格弱序的 4 条规则（comp(a,b) 表示比较结果）：① 反自反：comp(a,a) 必为 false；② 非对称：comp(a,b) 与 comp(b,a) 不能同时 true；③ 传递：comp(a,b) ∧ comp(b,c) ⇒ comp(a,c)；④ 等价传递：!comp(a,b) ∧ !comp(b,a) 视作等价，等价关系传递。违反任一条都是 UB（部分 libstdc++ 版本会真的崩溃）。
面试题"为什么 sort 是 introsort 而不是纯 quicksort?" — 答：纯 quicksort 最坏 O(n²)，恶意输入或已排序数据触发；introsort 用递归深度阈值检测后切换 heapsort 防御，合并三种算法的优点。

排序家族 ② — stable_sort / partial_sort / nth_element

三个高频场景 · REAL实际工程：日志的 P99 延迟统计

排序

stable_sort · partial_sort · nth_element

函数复杂度稳定?典型场景
sortO(n log n)否通用排序
stable_sortO(n log n) [若内存够] / O(n log² n)是多关键字排序
partial_sortO(n log k)否Top-K 元素
nth_elementO(n) 平均否找第 k 大 / 中位数
partial_sort_copyO(n log k)否不修改原数组的 Top-K
is_sortedO(n)—校验

三个高频场景

// ① stable_sort - 多关键字排序的关键
// 先按 score 降序，相同分数按 name 升序
std::stable_sort(users.begin(), users.end(),
    [](auto& a, auto& b){ return a.name < b.name; });
std::stable_sort(users.begin(), users.end(),
    [](auto& a, auto& b){ return a.score > b.score; });   // 主键，因稳定，name 顺序保留

// ② partial_sort - Top-K 元素到前面
std::partial_sort(v.begin(), v.begin() + 10, v.end());
// 前 10 个就是最小的 10 个（已排序）；后面顺序未定

// ③ nth_element - 找第 k 小（O(n) 平均，比 sort 快）
auto mid = v.begin() + v.size()/2;
std::nth_element(v.begin(), mid, v.end());
// 现在 *mid 是中位数；mid 左边都 ≤ *mid，右边都 ≥ *mid（左右内部不排序）

// 找第 k 小元素
std::nth_element(v.begin(), v.begin() + k, v.end());
int kth = v[k];

REAL实际工程：日志的 P99 延迟统计

// 错误做法：sort 全部 + 取 0.99*N 索引 → O(n log n)
// ✅ 正确：nth_element O(n)
double percentile(std::vector<double>& samples, double p) {
    auto idx = size_t(p * samples.size());
    std::nth_element(samples.begin(), samples.begin() + idx, samples.end());
    return samples[idx];
}
double p99 = percentile(latency_us, 0.99);   // 100M 样本: ~80ms vs sort ~1500ms
"如果只想要 Top-100 元素，怎么最快?" — 答：① partial_sort（O(n log 100)）；② 维护大小为 100 的最小堆遍历输入（O(n log 100) 但常数更小）；③ nth_element 找出第 100 大再 sort 前 100（O(n + 100 log 100)，最快）。三种各有适用。

二分查找族 — lower/upper/equal_range/binary_search

前提：序列已排序（按比较器排）。否则结果未定义。

查找⭐ 必考

二分查找族 — 易错的 4 兄弟

前提：序列已排序（按比较器排）。否则结果未定义。

函数含义返回复杂度
binary_search是否存在boolO(log n)
lower_bound(b,e,v)第一个 ≥ v 的位置iteratorO(log n)
upper_bound(b,e,v)第一个 > v 的位置iteratorO(log n)
equal_range(b,e,v){lower, upper} 一对pair<it,it>O(log n)

图解

典型用法

std::vector<int> v{1, 3, 3, 3, 5, 7, 9};

// 1. 是否存在
bool has = std::binary_search(v.begin(), v.end(), 3);   // true

// 2. 在已排序 vector 中插入并保持有序
auto pos = std::lower_bound(v.begin(), v.end(), 4);
v.insert(pos, 4);

// 3. 计算等于 v 的元素个数
auto [lo, hi] = std::equal_range(v.begin(), v.end(), 3);
auto count = hi - lo;            // O(log n)，比 std::count O(n) 快
陷阱：① lower_bound 返回的是插入位置，不一定指向 v；要先判断 it != end && *it == v 才算找到；② 在 std::list 上调用是 O(n)（迭代器只能 ++/--，二分跳转退化成线性扫描）；③ std::set::find() 自动 O(log n)，不要对 set 用 std::find（O(n)）。
面试问"map 和 lower_bound 哪个查找快?" — 答：都是 O(log n)；实际差异是 vector + lower_bound 缓存友好得多，N 不太大（< 10^5）时常优于 std::map。

堆算法 — make_heap / push_heap / pop_heap

典型流程 · REAL实战：固定大小 Top-K（流式）

堆

堆算法 — std::*_heap 系列

API

  • make_heap(b, e) — 把 [b,e) 整体调整为堆 (O(n))
  • push_heap(b, e) — 调用前已 push_back 新元素到末尾，调整上浮 (O(log n))
  • pop_heap(b, e) — 把堆顶 swap 到末尾，调整下沉 (O(log n))
  • sort_heap(b, e) — 把堆变成升序数组 (O(n log n))
  • is_heap(b, e) — 校验

默认大顶堆，传 std::greater<>{} 变小顶堆。

为什么不直接用 priority_queue?

  • priority_queue 没有迭代器，无法遍历或随机访问
  • priority_queue 只能 push 单元素，不能批量 build
  • 用 vector + std::*_heap 你能：
    ① 一次性 make_heap O(n)（vs N 次 push 是 O(n log n)）
    ② 仍能遍历底层 vector
    ③ 切换比较器更灵活

典型流程

std::vector<int> v{3, 1, 4, 1, 5, 9, 2, 6};

std::make_heap(v.begin(), v.end());          // O(n) — 整体建堆

v.push_back(8);                              // 先放末尾
std::push_heap(v.begin(), v.end());          // 再调整上浮

std::pop_heap(v.begin(), v.end());           // 堆顶移到末尾
int top = v.back();
v.pop_back();                                // 真正移除

REAL实战：固定大小 Top-K（流式）

// 维护 K 个最大元素：只需大小为 K 的小顶堆
class TopK {
    std::vector<int64_t> h_;
    size_t k_;
public:
    explicit TopK(size_t k) : k_(k) { h_.reserve(k); }
    void push(int64_t x) {
        if (h_.size() < k_) {
            h_.push_back(x);
            std::push_heap(h_.begin(), h_.end(), std::greater<>{});
        } else if (x > h_.front()) {                    // 比当前最小大才进入
            std::pop_heap(h_.begin(), h_.end(), std::greater<>{});
            h_.back() = x;
            std::push_heap(h_.begin(), h_.end(), std::greater<>{});
        }
    }
    std::vector<int64_t> snapshot() const { return h_; }   // 顺序未定但都是 top-k
};

// 100M 流入，K=100：复杂度 O(n log k)，内存 O(k)
"为什么 make_heap 是 O(n) 而 N 次 push_heap 是 O(n log n)?" — 答：从底向上 sift-down 时，大多数节点接近叶子（高度小），按 Σ h*N(h) 求和 = O(n)；而 push_heap 是从叶子向上 sift-up，每次都到根 O(log n)。这是经典分析题。

修改型算法 + erase-remove 惯用法

erase-remove 惯用法 · unique 的正确用法 · transform — 一对一映射

修改⭐

修改型算法 + erase-remove 惯用法

常用修改型算法

  • copy / copy_if / copy_n / copy_backward
  • move / move_backward — 用移动语义版本
  • transform — 一对一/二对一映射
  • fill / fill_n / generate / generate_n
  • replace / replace_if
  • remove / remove_if — 不真正删除，要配 erase
  • unique — 去除相邻重复（先 sort 才能全去重）
  • reverse / rotate / shuffle

关键陷阱：remove 不删除！

std::remove 把"要删的元素"的位置覆盖为后续的非删元素，返回新逻辑末尾。容器实际 size 不变。

必须配合 container.erase(new_end, container.end()) 才真正缩小。

erase-remove 惯用法

std::vector<int> v{1, 2, 3, 2, 4, 2, 5};

// ❌ 错误：v 大小没变，末尾留有未定义残留
std::remove(v.begin(), v.end(), 2);
// v = {1, 3, 4, 5, ?, ?, ?}  size 仍是 7

// ✅ 正确：erase-remove 惯用法
v.erase(std::remove(v.begin(), v.end(), 2), v.end());
// v = {1, 3, 4, 5}

// ✅ C++20 一行
std::erase(v, 2);
std::erase_if(v, [](int x){ return x % 2 == 0; });

unique 的正确用法

// unique 只移除"相邻"重复 — 不先排序则无效
std::vector<int> v{1, 2, 2, 3, 2, 1, 1};
v.erase(std::unique(v.begin(), v.end()), v.end());
// v = {1, 2, 3, 2, 1}  仅移除连续重复

// 全去重：先 sort + unique
std::sort(v.begin(), v.end());
v.erase(std::unique(v.begin(), v.end()), v.end());
// v = {1, 2, 3}

transform — 一对一映射

std::vector<int> src{1, 2, 3, 4};
std::vector<int> dst(src.size());
std::transform(src.begin(), src.end(), dst.begin(),
    [](int x){ return x * x; });        // dst = {1, 4, 9, 16}

// 二元 transform: 两个序列对应位置运算
std::vector<int> a{1,2,3}, b{10,20,30}, c(3);
std::transform(a.begin(), a.end(), b.begin(), c.begin(), std::plus<>{});
// c = {11, 22, 33}
erase-remove 是 C++03 时代的"老惯用法"，C++20 起优先用 std::erase / std::erase_if，更短更易读。

划分算法 — partition / stable_partition / partition_point

典型用法 · REAL实战：连接表中分离活跃/空闲连接

修改

划分算法 — partition 家族

API

  • partition(b, e, pred) — 把满足谓词的放前面，不稳定 O(n)
  • stable_partition — 同上但保持相对顺序，O(n) 若额外内存够
  • partition_point(b, e, pred) — 在已划分序列中找分界点（O(log n)）
  • is_partitioned — 检查是否已划分

为什么有用

  • quicksort 的 pivot 步骤就是 partition
  • nth_element 内部反复 partition 直到 pivot 落在第 k 位
  • 过滤 + 保留两种元素到不同区域

典型用法

std::vector<int> v{1, 5, 2, 8, 3, 9, 4};

// 把奇数放前，偶数放后
auto pivot = std::partition(v.begin(), v.end(),
    [](int x){ return x % 2 == 1; });
// v 可能 = {1, 5, 9, 3, 8, 2, 4}    奇数 [begin, pivot)，偶数 [pivot, end)

// stable_partition 保持原顺序
std::vector<int> v2{1, 5, 2, 8, 3, 9, 4};
std::stable_partition(v2.begin(), v2.end(),
    [](int x){ return x % 2 == 1; });
// v2 = {1, 5, 3, 9, 2, 8, 4}     奇数和偶数内部相对顺序保留

REAL实战：连接表中分离活跃/空闲连接

std::vector<Connection*> conns;
// 把活跃连接放前面，空闲放后面（稳定划分保持优先级顺序）
auto idle_begin = std::stable_partition(conns.begin(), conns.end(),
    [](const Connection* c){ return c->is_active(); });

// 现在：
//   [conns.begin(), idle_begin)  = 活跃
//   [idle_begin, conns.end())    = 空闲（可批量回收）

std::for_each(idle_begin, conns.end(),
    [](Connection* c){ c->close(); });
conns.erase(idle_begin, conns.end());
partition_point 之于 partitioned 序列 = lower_bound 之于 sorted 序列：都是 O(log n) 找分界点。

集合操作 — set_union / intersection / difference / merge

用法 · REAL实战：增量配置 diff

集合

集合操作算法（要求已排序）

API

  • merge(a1,a2,b1,b2,dst) — 合并两个已排序序列
  • inplace_merge(b, mid, e) — 把 [b,mid) 与 [mid,e) 合并为有序（原地）
  • set_union — 并集
  • set_intersection — 交集
  • set_difference — 差集 (A \ B)
  • set_symmetric_difference — 对称差 (A △ B)
  • includes — A ⊇ B 子集判定

共同前提

  • 所有输入都必须按同一比较器排序
  • 结果也按该比较器有序
  • 都是 O(n + m) 线性
  • 不要求输入是 set 容器，排序好的 vector 也行

用法

std::vector<int> a{1, 3, 5, 7};
std::vector<int> b{2, 3, 4, 7};
std::vector<int> out;

std::set_union(a.begin(), a.end(), b.begin(), b.end(), std::back_inserter(out));
// out = {1, 2, 3, 4, 5, 7}

out.clear();
std::set_intersection(a.begin(), a.end(), b.begin(), b.end(), std::back_inserter(out));
// out = {3, 7}

out.clear();
std::set_difference(a.begin(), a.end(), b.begin(), b.end(), std::back_inserter(out));
// out = {1, 5}      (a 中有但 b 中没有)

REAL实战：增量配置 diff

// 旧/新配置 - 已排序的 yang xpath 列表
std::vector<std::string> old_paths, new_paths;

std::vector<std::string> added, removed;
std::set_difference(new_paths.begin(), new_paths.end(),
                    old_paths.begin(), old_paths.end(),
                    std::back_inserter(added));    // 新增的
std::set_difference(old_paths.begin(), old_paths.end(),
                    new_paths.begin(), new_paths.end(),
                    std::back_inserter(removed));  // 删除的

// O(n+m) 替代两层循环 O(n*m) 的 diff
能讲出"set_* 算法不要求容器是 std::set，已排序 vector 也能用 — 而且更快（缓存友好）" 即加分。

数值算法 — accumulate / reduce / inner_product / iota

accumulate vs reduce（高频面试题） · 实用模式 · REAL实战：监控指标聚合

数值

数值算法 — accumulate / reduce / inner_product / iota

函数语义头文件
accumulate(b,e,init,op)左折叠 fold-left（顺序求值）<numeric>
reduce(b,e,init,op) (C++17)无序折叠（可并行 + 向量化）<numeric>
transform_reduce (C++17)map + reduce 组合<numeric>
inner_product(b1,e1,b2,init)点积 / 一一配对操作<numeric>
partial_sum / inclusive_scan / exclusive_scan前缀和<numeric>
adjacent_difference相邻差分<numeric>
iota(b,e,start)填充递增整数<numeric>

accumulate vs reduce（高频面试题）

std::vector<double> v(1'000'000);
// 顺序求和 - 必须按从左到右顺序（浮点不满足结合律）
double s1 = std::accumulate(v.begin(), v.end(), 0.0);

// 无序求和 - 允许并行，但需要 op 满足结合律
double s2 = std::reduce(std::execution::par_unseq,
                          v.begin(), v.end(), 0.0);
// 浮点 reduce 结果可能与 accumulate 略有不同（求和顺序不同）
陷阱：accumulate 默认 init 类型决定结果类型！
std::accumulate(v.begin(), v.end(), 0) 结果是 int，浮点数会被截断！
必须 std::accumulate(v.begin(), v.end(), 0.0) 或 0LL。

实用模式

// 平均值
double avg = std::reduce(v.begin(), v.end(), 0.0) / v.size();

// 平方和（map+reduce）
double ss = std::transform_reduce(v.begin(), v.end(), 0.0,
    std::plus<>{}, [](double x){ return x * x; });

// 字符串拼接
std::vector<std::string> ws{"hello", "world"};
std::string joined = std::accumulate(ws.begin(), ws.end(), std::string{},
    [](std::string acc, const std::string& w){ return acc + (acc.empty()?"":",") + w; });

// 前缀和
std::vector<int> sums(v.size());
std::partial_sum(v.begin(), v.end(), sums.begin());

// iota - 测试数据快速生成 / 索引序列
std::vector<int> idx(100);
std::iota(idx.begin(), idx.end(), 0);          // 0..99

REAL实战：监控指标聚合

// QPS 窗口统计
struct Sample { int64_t ts_us; double latency_us; uint32_t bytes; };

auto total_bytes = std::transform_reduce(

15 题完整答案速答详解

每题：关键词 + 原理 + 复杂度 + 加分点 · 30 秒可口述

面试⭐ 必背

15 题完整答案 — 速答模板

基础（送分，必须 3 秒答出）

Q1 · std::sort 用什么算法?

答：introsort（内省排序） = quicksort + heapsort + insertion sort 混合：开始用 quicksort（常数小、缓存友好），递归深度超过 2·log₂(n) 时切换 heapsort 防最坏 O(n²)，分段规模 < 16 用 insertion sort（小规模常数最小）。最终保证最坏 O(n log n)。

加分：三种算法分别防御的问题 — quicksort 缓存最好、heapsort 最坏可控、insertion 在 nearly-sorted 上接近 O(n)。std::stable_sort 则是 mergesort（保稳定）。

Q2 · sort 与 stable_sort 区别?

答：① 稳定性：sort 不保证相等元素相对顺序；stable_sort 保证。② 复杂度：sort 最坏 O(n log n)；stable_sort 内存足时 O(n log n)（mergesort），内存不足退化为 O(n log² n)。③ 内存：stable_sort 需要 O(n) 辅助内存。④ 常数：sort 常数更小。

工程：多关键字排序时优先 stable_sort 先排次要字段再排主要字段；或一次性 sort + lambda 同时比较多字段。

Q3 · lower_bound 与 upper_bound 区别?

答：都做二分，要求有序区间。

  • lower_bound(v, x)：第一个 ≥ x 的位置（找"插入点"或"是否存在 x"）
  • upper_bound(v, x)：第一个 > x 的位置
  • [lower, upper) = x 的全部等值区间 — 等价于 equal_range

判断 x 是否存在：it != end && *it == x，不要用 binary_search（它只返 bool，不给位置）。

Q4 · remove 真的会删除元素吗?

答：不会。remove / remove_if 是 erase-remove 惯用法 的前半段 — 它把不删的元素前移、返回新逻辑尾迭代器，容器 size 不变。必须配合 erase 才真删：

v.erase(std::remove(v.begin(), v.end(), 42), v.end());
// C++20:
std::erase(v, 42);            // 一句话完成

为什么这么设计：算法不持有容器、不能调整 size — 这是 STL"算法/容器分离"原则的体现。

Q5 · partial_sort 与 nth_element 各适合什么场景?

  • partial_sort(b, mid, e) — 把前 k 个最小元素 排好序 放在 [b, mid)，复杂度 O((n-k)·log k + k·log k)，内部基于堆。适合"Top K 还要排序"。
  • nth_element(b, nth, e) — 把第 k 小的元素放到 nth 位置，左边 ≤ nth、右边 ≥ nth，内部无序，平均 O(n)（Hoare 快速选择 + introselect 防最坏）。适合"只要找第 k 个 / 分位数"。

组合拳：要 Top K 排序 → nth_element + 局部 sort([b, b+k]) 比 partial_sort 还快（O(n + k·log k)），因为 nth_element 是线性的、partial_sort 每次还要维护堆。

中级（拉开差距）

Q6 · accumulate 和 reduce 区别?

  • accumulate（C++98）：严格顺序从左到右 (((a+b)+c)+d)。可指定 binary op，结果与顺序相关。
  • reduce（C++17）：允许 乱序 + 并行，op 必须满足 结合律 + 交换律，否则 UB（如浮点加法不严格结合 → 多次运行结果可能不同）。
  • transform_reduce：先 map 再 reduce，并行经典 pattern（如 dot product）。

关键：std::reduce(std::execution::par_unseq, v.begin(), v.end(), 0.0) 可线程并行 + SIMD 向量化，大数据求和比 accumulate 快 4-8×。

Q7 · 什么是严格弱序? 违反会怎样?

答：比较器必须满足 strict weak ordering：① 非自反 !(a<a)；② 反对称 a<b ⇒ !(b<a)；③ 传递 a<b ∧ b<c ⇒ a<c；④ 等价的传递性：!(a<b) ∧ !(b<a) 定义的"等价"也必须传递。

违反后果：std::sort 行为未定义 — 实测会死循环、段错误、越界写。常见错误：用 <= 替代 <（破坏 ①）；浮点比较带 NaN（NaN 与任何值都 !(a<b) && !(b<a) 但又不等价）。

排查：构造小数据 + assert(is_sorted(v.begin(), v.end(), cmp))。

Q8 · find Top-K 元素有几种方法? 哪种最快?

答：4 种方案：

方案复杂度适用场景
sort 全部O(n log n)k 接近 n、一次性
partial_sortO(n log k)要 Top K 排序、k 中等
nth_element + 局部 sortO(n + k log k)Top K 排序，最快
大小 k 的小顶堆遍历O(n log k)流式数据，唯一选择

非流式选 nth_element + sort；流式（数据无法全部装入内存）必须用堆 — 这是唯一能 O(1) 空间在线维护 Top K 的方案。

Q9 · stable_sort 为什么需要额外内存?

答：通常实现为 归并排序 (mergesort)，merge 步骤需要 O(n) 临时 buffer 存合并结果。libstdc++ 实现先 get_temporary_buffer 拿尽量大的辅助内存：

  • 拿到足够内存 → 标准 mergesort，O(n log n)。
  • 内存不足 → 退化为 in-place merge，O(n log² n)。

也是 STL 唯一会"内存换性能"的算法。常数比 std::sort 大 1.5-2 倍，不需要稳定性时优先 sort。

Q10 · set_* 算法的输入需要什么前提?

答：set_union / set_intersection / set_difference / set_symmetric_difference / includes / merge 这一族都要求 两个输入区间已按相同比较器有序。

  • 输出区间不能与输入重叠（除非 in-place 版本）；
  • 复杂度都是 O(n + m) 线性 — 比哈希集合的"先建集合再求交"快得多；
  • 可用于任何有序序列，不限于 std::set —— 这是 STL 算法的威力：函数操作的是迭代器范围，不是容器。

工程实例：两个 vector<int>（已排序）求交集，比塞进 std::set 再求交集快 5-10×（无堆分配、缓存友好）。日志去重、ID 集合运算、倒排索引归并都是这一族的天然场景。

进阶 / 体现深度（让面试官眼前一亮）

Q11 · make_heap 为什么是 O(n) 而不是 O(n log n)?

答：从最后一个非叶节点开始 自底向上 sift_down（不是自顶向下 push_heap）。第 i 层（从底数起）节点数约 n/2ⁱ，单次 sift_down 代价 O(i)，总和：

Σ (n / 2ⁱ) · i  ≈  n · Σ (i / 2ⁱ)  ≤  2n  =  O(n)

直觉：底层节点数多但下沉路径短、顶层路径长但节点少 — 加权和收敛到常数。

对比：逐个 push_heap 建堆是 O(n log n) — 每个新元素都可能上浮整棵树高，没有"短路径多、长路径少"的红利。

Q12 · STL 算法对 lambda / 函数指针 / std::function 性能差?

答：差距巨大（实测 std::sort 1M ints）：

  • lambda / functor — 每个 lambda 是唯一类型，编译器能完全内联，零开销抽象。最快（基准 1x）。
  • 函数指针 — 类型擦除少一层，但仍是间接调用（无 LTO 时无法内联）。慢 2-5×。
  • std::function — 类型擦除 + 可能小对象优化 + 虚表分发，慢 5-10×，还可能堆分配。

结论：STL 算法的 callable 参数永远首选 lambda / 仿函数；只在跨边界（plugin 接口）才用 std::function。std::sort + lambda 编出的汇编几乎和手写循环一样。

Q13 · par_unseq 不能有什么操作?

答：std::execution::par_unseq = 并行 + 向量化交错执行。lambda body 内禁止：

  1. 任何同步原语：mutex、lock_guard、condition_variable — 同一线程内可能交错执行不同迭代，会自死锁（同一线程递归锁不可重入 mutex）。
  2. 抛异常 — 异常会直接调 std::terminate。
  3. 不可重入的全局状态：随机数生成器、std::cout、IO。
  4. 依赖执行顺序的副作用 — 反正乱序，别假设顺序。

par（仅并行不向量化）则允许 mutex。seq 是顺序、纯单线程。简单经验：lambda body 纯函数最安全。

Q14 · 怎么用 STL 算法实现 LRU 中"移到头部"?

答：经典 LRU = std::list<pair<K, V>> + std::unordered_map<K, list::iterator>。"访问后移到头部"用 list::splice（节点指针操作，O(1) 且迭代器不失效）：

void touch(K key) {
    auto it = map_[key];
    list_.splice(list_.begin(), list_, it);   // 节点搬到 begin，O(1)
}

不要用 erase + push_front — 那会让 map 里存的 iterator 失效，必须重新插入。splice 是 list 的杀手锏，也是 list 唯一在 vector 上无法 O(1) 模拟的操作之一 — 这就是 LRU 必须用 list 的根本原因。

Q15 · 透明 functor (less<>) 与 less<int> 区别?

  • std::less<int>：固定比较两个 int，传入其他类型要先构造 int。
  • std::less<>（C++14，透明仿函数）：模板化的 operator()(auto&&, auto&&)，不强制两侧同型，能直接比较 string vs string_view、int vs long。

关键应用 — 异构查找 (heterogeneous lookup, C++14)：

std::map<std::string, V, std::less<>> m;
m.find(std::string_view{"abc"});  // ✅ 不必构造 std::string
m.find("abc");                    // ✅ 不必构造 std::string

每次 lookup 省一次堆分配 — 热点路径上能减少几十 % 的开销。unordered_xxx 版本要求自定义 hash + equal 都标 using is_transparent = void; 才生效。STL find / lower_bound / sort 默认就是透明 lambda → 永远首选 std::less<>、std::greater<>，几乎没有理由用类型固化版本。

背诵口诀：每题"一句话定义 + 一句话原理 + 一个数字/关键词"。例如 Q1：「introsort = quicksort + heapsort + insertion sort 三合一；递归深度 2·log₂n 是 quicksort 切 heapsort 的阈值；< 16 切 insertion」。三段式 30 秒讲完，面试官立刻给你打满分。
